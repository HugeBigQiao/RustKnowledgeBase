# 枚举

Rust 枚举不仅是 C 风格的命名常量，还可以包含不同类型的数据，类似于 ML/Haskell 中的**代数数据类型**或**总和类型**。只要值可能代表多种事物，枚举就很有用。"代价"是必须通过模式匹配来安全访问数据。

## 一、C 风格枚举

```rust
enum Ordering {
    Less,
    Equal,
    Greater,
}

fn compare(n: i32, m: i32) -> Ordering {
    if n < m { Ordering::Less }
    else if n > m { Ordering::Greater }
    else { Ordering::Equal }
}
```

### 自定义判别值

```rust
enum HttpStatus {
    Ok = 200,
    NotModified = 304,
    NotFound = 404,
}

assert_eq!(HttpStatus::Ok as i32, 200);
```

默认从 0 开始分配整数。可转换为整数，但**不能反向转换**（必须自己编写安全转换）。

### 内存

Rust 使用可容纳所有变体的最小整数存储 C 风格枚举，通常 1 字节。可用 `#[repr]` 覆盖。

```rust
assert_eq!(std::mem::size_of::<Ordering>(), 1);
assert_eq!(std::mem::size_of::<HttpStatus>(), 2);  // 404 不适合 u8
```

### 方法与派生

```rust
#[derive(Copy, Clone, Debug, PartialEq, Eq)]
enum TimeUnit { Seconds, Minutes, Hours, Days, Months, Years }

impl TimeUnit {
    fn plural(self) -> &'static str {
        match self {
            TimeUnit::Seconds => "seconds",
            TimeUnit::Minutes => "minutes",
            // ...
        }
    }
    fn singular(self) -> &'static str {
        self.plural().trim_end_matches('s')
    }
}
```

## 二、带数据的枚举

Rust 枚举有三种变体形式，对应三种结构体：

### 元组型变体

```rust
enum RoughTime {
    InThePast(TimeUnit, u32),
    JustNow,
    InTheFuture(TimeUnit, u32),
}

let four_score = RoughTime::InThePast(TimeUnit::Years, 87);
let three_hours = RoughTime::InTheFuture(TimeUnit::Hours, 3);
```

### 结构体型变体

```rust
enum Shape {
    Sphere { center: Point3d, radius: f32 },
    Cuboid { corner1: Point3d, corner2: Point3d },
}

let unit_sphere = Shape::Sphere { center: ORIGIN, radius: 1.0 };
```

### 三种变体混合

```rust
enum RelationshipStatus {
    Single,                                  // 单元型变体
    InARelationship,
    ItsComplicated(Option<String>),           // 元组型变体
    ItsExtremelyComplicated {                 // 结构体型变体
        car: DifferentialEquation,
        cdr: EarlyModernistPoem,
    },
}
```

## 三、内存中的枚举

枚举使用**标签**（小整数）+ 足以容纳最大变体的内存块：

- 标签由 Rust 内部使用，区分值由哪个构造器创建
- Rust 不承诺具体布局，为未来优化留空间

特殊优化：当 `T` 是引用或 `Box` 等非空指针时，`Option<T>` 可以省略标签（0 表示 `None`，非零表示 `Some`），与 C/C++ 空指针一样高效但类型安全。

## 四、富数据结构表示

枚举特别适合表示 JSON、AST 等递归数据结构：

```rust
use std::collections::HashMap;

enum Json {
    Null,
    Boolean(bool),
    Number(f64),
    String(String),
    Array(Vec<Json>),
    Object(Box<HashMap<String, Json>>),  // Box 让 Json 保持紧凑（4 机器字）
}
```

用 Rust 表示这种结构仅需 8 行代码。C++ 需要手动管理联合体、标签、构造/析构函数，几十行起步。

## 五、泛型枚举

```rust
enum Option<T> {
    None,
    Some(T),
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

### 泛型树形结构示例

```rust
enum BinaryTree<T> {
    Empty,
    NonEmpty(Box<TreeNode<T>>),
}

struct TreeNode<T> {
    element: T,
    left: BinaryTree<T>,
    right: BinaryTree<T>,
}

// 构造
use self::BinaryTree::*;
let tree = NonEmpty(Box::new(TreeNode {
    element: "Saturn",
    left: mars_tree,
    right: uranus_tree,
}));
```

与 `Option<Box<T>>` 一样，Rust 对 `BinaryTree<T>` 也会省略标签，值只占一个机器字。

## 六、通过模式匹配访问枚举

枚举数据**只能通过模式匹配安全访问**，不能直接访问字段：

```rust
// 错误：shape.radius —— Shape 上没有 radius 字段

// 正确：match + 模式
fn rough_time_to_english(rt: RoughTime) -> String {
    match rt {
        RoughTime::InThePast(units, count) =>
            format!("{} {} ago", count, units.plural()),
        RoughTime::JustNow =>
            format!("just now"),
        RoughTime::InTheFuture(unit, 1) =>
            format!("a {} from now", unit.singular()),
        RoughTime::InTheFuture(units, count) =>
            format!("{} {} from now", count, units.plural()),
    }
}
```

`match` 的分支顺序很重要——更具体的模式必须放在前面，否则 Rust 会警告"无法抵达的模式"。

## 七、模式匹配语言速览

`match` 不仅能用于枚举，还能匹配任意类型：

| 模式类型 | 示例 | 说明 |
|---------|------|------|
| 字面量 | `100`, `"name"` | 精确匹配 |
| 范围 | `0 ..= 100`, `'a' ..= 'k'` | 区间匹配 |
| 通配符 | `_` | 匹配任何值并忽略 |
| 变量 | `name`, `mut count` | 绑定值到变量 |
| 引用 | `ref field`, `ref mut field` | 借用而非移动 |
| `@` 绑定 | `val @ 0 ..= 99` | 同时绑定和匹配 |
| `\|` 多模式 | `'a' \| 'A'` | 匹配任一 |
| 守卫 | `x if x * x <= r2` | 附加条件 |

```rust
match meadow.count_rabbits() {
    0 => {},                      // 字面量
    1 => println!("A rabbit..."), // 字面量
    n => println!("{} rabbits", n),  // 变量绑定
}

// 字符串字面量
let calendar = match settings.get_string("calendar") {
    "gregorian" => Calendar::Gregorian,
    "chinese"   => Calendar::Chinese,
    other       => return parse_error("calendar", other),
};

// 通配符
let caption = match photo.tagged_pet() {
    Pet::Tyrannosaur => "RRRAAAAAHHHHHH",
    _ => "I'm cute, love me",
};
```

> 模式匹配的完整深入讲解（元组型/结构体型/数组型/切片型/引用型模式、`@` 绑定、守卫等）参见 [模式匹配深入.md](模式匹配深入.md)。

## 速记要点

- 枚举是**类型安全的联合体**，变体可以有数据（元组型/结构体型）或无数据（单元型）
- 内存布局：标签 + 变体数据的最大空间；非空指针类型可省略标签
- 枚举数据只能通过 **`match` + 模式**安全访问
- 泛型枚举（`Option<T>`、`Result<T, E>`）是 Rust 错误处理和可选值的基础
- 枚举特别适合表示**递归数据结构**、**多态类型**（如 JSON AST）
- `match` 必须穷尽所有可能，通配符 `_` 或变量绑定用作后备分支
