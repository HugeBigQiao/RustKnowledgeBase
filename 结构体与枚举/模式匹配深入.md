# 模式匹配深入

Rust 模式是一种"针对数据的正则表达式"。它们用于测试值是否具有特定形态，并将字段一次性提取到局部变量中。模式不仅是 `match` 的专属，也可以用于 `let`、函数参数、`for` 和闭包参数。

## 一、模式语言速览

| 模式类型 | 示例 | 说明 |
|---------|------|------|
| 字面量 | `100`, `"name"` | 精确匹配 |
| 范围 | `0 ..= 100`, `'a' ..= 'k'` | 闭区间匹配 |
| 通配符 | `_` | 匹配任何值并忽略 |
| 变量 | `name`, `mut count` | 绑定到新局部变量 |
| `ref` 引用 | `ref field`, `ref mut field` | 借用而非移动 |
| `&` 匹配引用 | `&value`, `&(k, v)` | 匹配目标为引用 |
| `@` 绑定 | `val @ 0 ..= 99` | 同时绑定和匹配 |
| `\|` 多模式 | `'a' \| 'A'` | 匹配任一 |
| 守卫 | `x if x * x <= r2` | 附加条件（仅 match） |

## 二、元组型模式

匹配元组并将元素绑定到变量：

```rust
fn describe_point(x: i32, y: i32) -> &'static str {
    use std::cmp::Ordering::*;
    match (x.cmp(&0), y.cmp(&0)) {
        (Equal, Equal) => "at the origin",
        (_, Equal)     => "on the x axis",
        (Equal, _)     => "on the y axis",
        (Greater, Greater) => "in the first quadrant",
        (Less, Greater)    => "in the second quadrant",
        _ => "somewhere else",
    }
}
```

## 三、结构体型模式

```rust
struct Point { x: i32, y: i32 }

match balloon.location {
    Point { x: 0, y: height } =>   // 精确匹配 x=0，y 绑定到 height
        println!("straight up {} meters", height),
    Point { x, y } =>              // 简写：绑定 x, y
        println!("at ({}m, {}m)", x, y),
}
```

### 忽略不关心的字段

```rust
// 冗长写法
Some(Account { name, language, id: _, status: _, .. })

// 最佳写法
Some(Account { name, language, .. }) =>
    language.show_custom_greeting(name),
```

## 四、数组型模式与切片型模式

```rust
// 数组型模式
fn hsl_to_rgb(hsl: [u8; 3]) -> [u8; 3] {
    match hsl {
        [_, _, 0]   => [0, 0, 0],
        [_, _, 255] => [255, 255, 255],
        // ...
    }
}

// 切片型模式（可变长度）
fn greet_people(names: &[&str]) {
    match names {
        []       => println!("Hello, nobody."),
        [a]      => println!("Hello, {}.", a),
        [a, b]   => println!("Hello, {} and {}.", a, b),
        [a, .., b] => println!("Hello, everyone from {} to {}.", a, b),
    }
}
```

## 五、引用型模式

### ref 模式——借用而非移动

没有 `ref` 时，匹配会**移动**非 Copy 类型的值：

```rust
// 错误：name 和 language 移动后 account 不再完整
match account {
    Account { name, language, .. } => {
        ui.greet(&name, &language);
        ui.show_settings(&account);  // 错误：account 已被部分移动
    }
}
```

使用 `ref` 来借用：

```rust
match account {
    Account { ref name, ref language, .. } => {
        ui.greet(name, language);           // name, language 是引用
        ui.show_settings(&account);         // 正确！
    }
}
```

`ref mut` 借入可变引用：

```rust
match line_result {
    Err(ref err) => log_error(err),     // err: &Error
    Ok(ref mut line) => {
        trim_comments(line);            // 就地修改
        handle(line);
    }
}
```

### & 模式——匹配引用

```rust
match sphere.center() {
    &Point3d { x, y, z } => ...    // 匹配 &Point3d，绑定 x, y, z
}
```

表达式中的 `&` 创建引用，模式中的 `&` 匹配引用：

```rust
match chars.peek() {
    Some(&c) => println!("coming up: {:?}", c),  // c 是 char 的副本
    None => println!("end of chars"),
}
```

> 不能通过共享引用获得可变的访问权限；不能将值从引用中移动出去。若字段不是 Copy 型可结合 `ref`：`Some(&Car { ref engine, .. })`。

## 六、匹配守卫

附加条件：模式匹配成功但不满足守卫时继续匹配下一个分支：

```rust
match point_to_hex(click) {
    None =>
        Err("That's not a game space."),
    Some(hex) if hex == current_hex =>
        Err("You are already there!"),
    Some(hex) =>
        Ok(hex),
}
```

> 注意：模式中的标识符引入的是**新变量**，会遮蔽外部同名变量。匹配守卫中的引用指向外部变量。

## 七、多模式匹配（|）

```rust
let at_end = match chars.peek() {
    Some(&'\r' | &'\n') | None => true,
    _ => false,
};

match next_char {
    '0'..='9'                => self.read_number(),
    'a'..='z' | 'A'..='Z'    => self.read_word(),
    ' ' | '\t' | '\n'        => self.skip_whitespace(),
    _                        => self.handle_punctuation(),
}
```

## 八、@ 绑定

将匹配的整个值与子模式同时绑定：

```rust
// 不善的写法：先解构再重建
match self.get_selection() {
    Shape::Rect(top_left, bottom_right) =>
        optimized_paint(&Shape::Rect(top_left, bottom_right)),
    other_shape => paint_outline(other_shape.get_outline()),
}

// 优雅的写法：@ 绑定整个值
match self.get_selection() {
    rect @ Shape::Rect(..) => optimized_paint(&rect),
    other_shape => paint_outline(other_shape.get_outline()),
}
```

结合范围使用：

```rust
match chars.next() {
    Some(digit @ '0'..='9') => read_number(digit, chars),
    // ...
}
```

## 九、模式可用位置

### let 语句

```rust
let Track { album, track_number, title, .. } = song;
```

### 函数参数

```rust
fn distance_to((x, y): (f64, f64)) -> f64 { (x.powi(2) + y.powi(2)).sqrt() }
```

### for 循环

```rust
for (id, document) in &cache_map {
    println!("Document #{}: {}", id, document.title);
}
```

### 闭包参数

```rust
let sum = numbers.fold(0, |a, &num| a + num);  // 自动解引用
```

### if let 表达式

```rust
if let RoughTime::InTheFuture(_, _) = user.date_of_birth() {
    user.set_time_traveler(true);
}

if let Some(document) = cache_map.get(&id) {
    return send_cached_response(document);
}
```

### while let 表达式

```rust
while let Err(err) = present_cheesy_anti_robot_task() {
    log_robot_attempt(err);
}

while let Some(_) = lines.peek() {
    read_paragraph(&mut lines);
}
```

## 十、不可反驳模式与可反驳模式

| 类型 | 说明 | 可用位置 |
|------|------|---------|
| **不可反驳模式** | 总是匹配成功 | `let`、函数参数、`for`、闭包参数 |
| **可反驳模式** | 可能匹配失败 | `match` 分支、`if let`、`while let` |

```rust
// 不可反驳：Point3d { x, y, z } 匹配所有 Point3d
let Point3d { x, y, z } = point;

// 可反驳：Ok(x) 不匹配 Err
if let Ok(x) = result { ... }
```

## 速记要点

- 模式是"反向的表达式"——表达式构建值，模式消耗/解构值
- `ref` 借用（不移动），`ref mut` 可变借用
- `&` 模式匹配引用目标
- `@` 绑定整个值同时解构子部分
- `..` 忽略其余字段/元素
- `|` 匹配多个可能模式
- `if` 守卫在模式匹配之后附加条件
- `match` 必须穷尽所有可能，通配符 `_` 作为后备
