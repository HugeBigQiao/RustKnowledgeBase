# 结构体

Rust 中的结构体将多个不同类型的值组合成单一值，类似于 C/C++ 的 `struct`、Python 的类和 JavaScript 的对象。Rust 有 3 种结构体类型：**具名字段型结构体**、**元组型结构体**和**单元型结构体**。

## 一、具名字段型结构体

最常用的结构体形式，每个字段都有名称：

```rust
struct GrayscaleMap {
    pixels: Vec<u8>,
    size: (usize, usize),
}
```

### 构造与访问

```rust
let width = 1024;
let height = 576;
let image = GrayscaleMap {
    pixels: vec![0; width * height],
    size: (width, height),
};

// 同名变量简写
fn new_map(size: (usize, usize), pixels: Vec<u8>) -> GrayscaleMap {
    GrayscaleMap { pixels, size }   // 等价于 pixels: pixels, size: size
}

assert_eq!(image.size, (1024, 576));
assert_eq!(image.pixels.len(), 1024 * 576);
```

### 结构体更新语法

```rust
struct Broom { name: String, height: u32, health: u32, position: (f32, f32, f32) }

fn chop(b: Broom) -> (Broom, Broom) {
    let mut broom1 = Broom { height: b.height / 2, .. b };
    let mut broom2 = Broom { name: broom1.name.clone(), .. broom1 };
    broom1.name.push_str(" I");
    broom2.name.push_str(" II");
    (broom1, broom2)
}
```

`.. b` 表示其余字段从 `b` 中获取。注意非 `Copy` 类型（如 `String`）会随 `..` 移动所有权。

### 可见性

```rust
pub struct GrayscaleMap {
    pub pixels: Vec<u8>,    // 公开
    size: (usize, usize),   // 私有——即使结构体是 pub
}
```

## 二、元组型结构体

像具名元组，字段按位置访问：

```rust
struct Bounds(usize, usize);

let image_bounds = Bounds(1024, 768);
assert_eq!(image_bounds.0 * image_bounds.1, 786432);
```

常用于创建 **newtype**（新类型）模式：

```rust
struct Ascii(Vec<u8>);  // 比裸露的 Vec<u8> 更有类型安全性
```

## 三、单元型结构体

没有字段的结构体，不占内存：

```rust
struct Onesuch;

let o = Onesuch;
```

典型用途：`RangeFull`（`..` 运算符）、配合特型使用（作为标记类型）。

## 四、结构体布局

Rust 不承诺字段在内存中的具体排列顺序，但保证字段值直接存储在结构体的内存块中（不像 Java/Python 将字段放在单独的堆块中）。可使用 `#[repr(C)]` 强制 C 兼容布局。

## 五、impl 定义方法

方法定义在单独的 `impl` 块中：

```rust
pub struct Queue {
    older: Vec<char>,
    younger: Vec<char>,
}

impl Queue {
    /// 推入队列尾部
    pub fn push(&mut self, c: char) {
        self.younger.push(c);
    }

    /// 从队列前端弹出
    pub fn pop(&mut self) -> Option<char> {
        if self.older.is_empty() {
            if self.younger.is_empty() { return None; }
            use std::mem::swap;
            swap(&mut self.older, &mut self.younger);
            self.older.reverse();
        }
        self.older.pop()
    }

    /// 按共享引用获取 self
    pub fn is_empty(&self) -> bool {
        self.older.is_empty() && self.younger.is_empty()
    }

    /// 按值获取 self——消耗所有权
    pub fn split(self) -> (Vec<char>, Vec<char>) {
        (self.older, self.younger)
    }
}
```

### self 参数速查

| 写法 | 含义 | 调用时隐式操作 |
|------|------|---------------|
| `self` | 消耗所有权 | 移动 |
| `&self` | 共享引用 | 共享借用 |
| `&mut self` | 可变引用 | 可变借用 |
| `self: Box<Self>` | 拥有型指针 | 较少使用 |
| `self: Rc<Self>` | 引用计数指针 | 特定场景 |

### 类型关联函数（不取 self）

```rust
impl Queue {
    pub fn new() -> Queue {
        Queue { older: Vec::new(), younger: Vec::new() }
    }
}

let mut q = Queue::new();   // Queue::new 是关联函数，不是方法
```

## 七、泛型结构体

```rust
pub struct Queue<T> {
    older: Vec<T>,
    younger: Vec<T>,
}

impl<T> Queue<T> {
    pub fn new() -> Self {
        Queue { older: Vec::new(), younger: Vec::new() }
    }
    pub fn push(&mut self, t: T) {
        self.younger.push(t);
    }
}

// 特定类型的专属 impl
impl Queue<f64> {
    fn sum(&self) -> f64 { /* ... */ }
}
```

## 八、带生命周期参数的泛型结构体

```rust
struct Extrema<'elt> {
    greatest: &'elt i32,
    least: &'elt i32,
}

fn find_extrema(slice: &[i32]) -> Extrema {
    // 生命周期可省略：最常见的场景 Rust 自动推断
    let mut greatest = &slice[0];
    let mut least = &slice[0];
    for i in 1..slice.len() {
        if slice[i] < *least    { least    = &slice[i]; }
        if slice[i] > *greatest { greatest = &slice[i]; }
    }
    Extrema { greatest, least }
}
```

## 九、带常量参数的泛型结构体

```rust
/// N - 1 次多项式
struct Polynomial<const N: usize> {
    coefficients: [f64; N],
}

impl<const N: usize> Polynomial<N> {
    fn new(coefficients: [f64; N]) -> Polynomial<N> {
        Polynomial { coefficients }
    }

    fn eval(&self, x: f64) -> f64 {
        let mut sum = 0.0;
        for i in (0..N).rev() {
            sum = self.coefficients[i] + x * sum;
        }
        sum
    }
}

let sine_poly = Polynomial::new([0.0, 1.0, 0.0, -1.0/6.0, 0.0, 1.0/120.0]);
assert!((sine_poly.eval(std::f64::consts::FRAC_PI_2) - 1.).abs() < 0.005);
```

> 常量泛型参数可以是任意整数类型、`char` 或 `bool`，目前不能在类型中使用受限的运算（如 `N + 1`）。

参数顺序：生命周期参数 → 类型参数 → `const` 参数。

## 十、派生公共特型

```rust
#[derive(Copy, Clone, Debug, PartialEq)]
struct Point { x: f64, y: f64 }
```

常用派生特型：`Copy`、`Clone`、`Debug`、`PartialEq`、`Eq`、`PartialOrd`、`Hash`、`Default`。

派生要求所有字段都实现了该特型。

## 十一、内部可变性（Cell / RefCell）

当需要在不可变值中获得可变数据时使用：

### Cell<T>

适用于 `Copy` 类型，通过 `get()` 和 `set()` 在不可变引用上操作：

```rust
use std::cell::Cell;

pub struct SpiderRobot {
    hardware_error_count: Cell<u32>,
}

impl SpiderRobot {
    pub fn add_hardware_error(&self) {
        let n = self.hardware_error_count.get();
        self.hardware_error_count.set(n + 1);
    }
}
```

### RefCell<T>

允许在不可变引用上借出可变引用，运行时检查借用规则：

```rust
use std::cell::RefCell;

pub struct SpiderRobot {
    log_file: RefCell<File>,
}

impl SpiderRobot {
    pub fn log(&self, message: &str) {
        let mut file = self.log_file.borrow_mut();  // 运行时可变借用
        writeln!(file, "{}", message).unwrap();
    }
}
```

| 方法 | 说明 |
|------|------|
| `borrow()` | 返回 `Ref<T>`（共享引用），若已有可变借用则 panic |
| `borrow_mut()` | 返回 `RefMut<T>`（可变引用），若有任何借用则 panic |
| `try_borrow()` / `try_borrow_mut()` | 返回 `Result`，不 panic |

> Cell 和 RefCell 不是线程安全的。线程间内部可变性参见 [并发与异步/共享可变状态.md](../并发与异步/共享可变状态.md)。
