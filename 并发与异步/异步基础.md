# Rust 异步编程基础

## 一、概述

异步任务类似于线程，但创建更快、内存开销更少（约为线程的 1/100），在单个程序中同时运行数十万个异步任务是可行的。

**与多线程代码核心区别**：阻塞操作（如 I/O）改为 `.await` 方式，让线程在等待时执行其他任务。

> 进阶话题（Pin/Unpin、Stream、执行器原理、异步互斥锁等）请参见：[异步深入](异步深入.md)。

## 二、`Future` 特型

### 2.1 定义

```rust
trait Future {
    type Output;
    fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output>;
}

enum Poll<T> {
    Ready(T),
    Pending,
}
```

- `Future` 代表可以测试其完成情况的操作
- `poll` 从不等待：操作完成返回 `Poll::Ready(output)`，否则返回 `Pending`
- 如果值得再次轮询，`Future` 承诺通过 `Context` 中的 `waker` 回调通知

### 2.2 皮纳塔模型

对 `Future` 唯一能做的就是通过 poll 轮询它，直到值"掉"出来——就像敲打皮纳塔一样。

### 2.3 异步版函数签名

```rust
// 同步版
fn read_to_string(&mut self, buf: &mut String) -> std::io::Result<usize>;

// 异步版
fn read_to_string<'a>(&'a mut self, buf: &'a mut String)
    -> impl Future<Output = Result<usize>> + 'a;
```

异步版本返回 `Future`，调用时不会立即执行任何操作，只是构建并返回 `Future`。

## 三、异步函数与 `await`

### 3.1 `async fn` 语法

```rust
use async_std::io::prelude::*;
use async_std::net;

async fn cheapo_request(host: &str, port: u16, path: &str)
                        -> std::io::Result<String>
{
    let mut socket = net::TcpStream::connect((host, port)).await?;
    let request = format!("GET {} HTTP/1.1\r\nHost: {}\r\n\r\n", path, host);
    socket.write_all(request.as_bytes()).await?;
    socket.shutdown(net::Shutdown::Write)?;
    let mut response = String::new();
    socket.read_to_string(&mut response).await?;
    Ok(response)
}
```

**核心要点**：
- `async fn f(...) -> T` 返回的是 `Future<Output=T>`，而非 `T`
- 调用异步函数时函数体**不立即执行**，只返回 `Future`
- `await` 表达式会获取 `Future` 的所有权并轮询，若未就绪则返回 `Poll::Pending` 并将控制权交还调用者
- 下一次轮询时从暂停点**恢复执行**（而非从头开始）

### 3.2 `await` 只能在异步函数中使用

`await` 表达式的暂停/恢复能力是异步函数独有的，只能在异步函数或异步块中使用。

### 3.3 异步任务切换时机

从一个异步任务切换到另一个只在 `await` 表达式处发生，且仅当等待的 `Future` 返回 `Poll::Pending` 时。这意味着长时间运行的计算会阻塞其他任务（解决方案见 [异步深入](异步深入.md)）。

## 四、同步调用异步：`block_on`

```rust
fn main() -> std::io::Result<()> {
    use async_std::task;
    let response = task::block_on(cheapo_request("example.com", 80, "/"))?;
    println!("{}", response);
    Ok(())
}
```

- `block_on` 是异步世界到同步世界的适配器
- 接受 `Future` 并反复轮询直到生成值
- **不应在异步函数中使用** `block_on`（它会阻塞整个线程）

## 五、异步任务：`spawn_local` 与 `spawn`

### 5.1 `spawn_local` — 在当前线程运行

```rust
pub async fn many_requests(requests: Vec<(String, u16, String)>)
                           -> Vec<std::io::Result<String>>
{
    let mut handles = vec![];
    for (host, port, path) in requests {
        handles.push(task::spawn_local(async move {
            cheapo_request(&host, port, &path).await
        }));
    }
    let mut results = vec![];
    for handle in handles {
        results.push(handle.await);
    }
    results
}
```

- `spawn_local` 将 `Future` 添加到当前线程的轮询池
- 返回 `JoinHandle`（本身是 `Future`）
- 要求 `Future: 'static`

### 5.2 `spawn` — 在线程池运行

```rust
task::spawn(async move { ... })
```

- 在工作线程池中轮询 `Future`
- 要求 `Future: Send + 'static`（因为可能在不同线程间转移）
- 异步调用可能在一个线程开始、阻塞在 `await`、在另一个线程恢复

## 六、异步块

```rust
let serve_one = async {
    let listener = net::TcpListener::bind("localhost:8087").await?;
    let (mut socket, _addr) = listener.accept().await?;
    // 在 socket 上与客户端对话
};
```

- 异步块返回其最后一个表达式值的 `Future`
- `async move` 块获取捕获变量的所有权（类似 `move` 闭包）
- `?` 运算符和 `return` 只在块中返回
- 异步块无法显式指定返回类型，可能导致类型推断问题

### 6.1 解决异步块类型推断

```rust
let future = async {
    // ...
    Ok::<(), std::io::Error>(())
};
```

### 6.2 从异步块构建异步函数

```rust
fn cheapo_request(host: &str, port: u16, path: &str)
    -> impl Future<Output = io::Result<String>> + 'static
{
    let host = host.to_string();
    let path = path.to_string();
    async move { /* 使用 &*host, port, path */ }
}
```
