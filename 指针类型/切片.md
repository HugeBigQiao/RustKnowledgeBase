# Rust 切片类型

切片 `&[T]` 是一种**不拥有数据所有权**的指针类型，它提供了对连续序列（如数组或向量的一部分）的"视图"。切片在内存中是一个**胖指针**，包含指向数据的指针和长度。与 `Vec<T>` 和数组不同，切片本身不分配内存，只是借用已有的数据。

## 一、速览

| 特性 | 说明 |
| :--- | :--- |
| 存储位置 | 栈（胖指针） |
| 长度 | 运行时已知，但不可变 |
| 元素类型 | 必须全部相同 |
| 是否拥有数据 | **否**，只是借用 |
| 默认值 | 无（通常由引用获得） |

## 二、切片的本质：胖指针

切片在内存中占用两个 `usize` 大小：
    - 一个指向数据起始地址的指针
    - 一个表示数据元素个数的长度

这种双字段结构使它成为"胖指针"，区别于普通引用只存一个地址。

```rust
    let arr = [10, 20, 30, 40, 50];
    let slice: &[i32] = &arr[1..4];  // 指向 arr 索引 1..4 的视图
    // slice 内部存储：指向 arr[1] 的指针 + 长度 3
```

创建切片**不会复制数据**，只是重新解释指针和长度，因此效率极高。

### 胖指针与 DST（动态大小类型）

`[T]` 本身是一个**动态大小类型**（Dynamically Sized Type, DST），编译时不知道其大小。因此 `[T]` 不能直接作为局部变量，只能通过胖指针 `&[T]` 或 `Box<[T]>` 来使用。同理，`str` 也是 DST，必须通过 `&str` 或 `Box<str>` 引用。

```rust
    // 错误：编译时不知道 [i32] 的大小
    // let bad: [i32] = [1, 2, 3];

    // 正确：通过胖指针引用
    let good: &[i32] = &[1, 2, 3];
```

`Sized` trait 标记编译时大小已知的类型。大多数类型实现了 `Sized`，但 `[T]`、`str` 和特型对象没有。Rust 在泛型中默认要求 `T: Sized`，可以用 `?Sized` 放宽此限制。

## 三、创建切片

切片只能通过借用已有集合来创建，不能直接构造独立的切片变量。

```rust
    // 从数组创建切片
    let arr = [1, 2, 3, 4, 5];
    let s1: &[i32] = &arr;         // 完整切片
    let s2 = &arr[0..3];           // 部分切片 [1,2,3]
    let s3 = &arr[..3];            // 从开头到索引3（不含）
    let s4 = &arr[2..];            // 从索引2到结尾
    let s5 = &arr[..];             // 整个数组的切片

    // 从 Vec 创建切片
    let v = vec![10, 20, 30];
    let s6: &[i32] = &v;           // 自动解引用为切片
    let s7 = &v[1..];              // 部分切片

    // 字符串切片 &str 也是切片的一种
    let string = String::from("hello");
    let s8: &str = &string;        // 完整的字符串切片
    let s9 = &string[0..2];        // "he"
```

## 四、切片的方法

切片提供了丰富的方法，大部分与数组和 `Vec` 共享，因为它们都实现了 `std::slice::SliceIndex` trait。

```rust
    let data = [3, 1, 4, 1, 5, 9, 2];
    let s: &[i32] = &data;

    // 长度和空检查
    s.len();         // 7
    s.is_empty();    // false

    // 访问首尾元素
    s.first();       // Some(&3)
    s.last();        // Some(&2)

    // 按索引安全访问
    s.get(2);        // Some(&4)
    s.get(100);      // None

    // 查找
    s.contains(&4);  // true
    s.starts_with(&[3, 1]); // true

    // 遍历（切片实现了 IntoIterator trait）
    for elem in s {
        println!("{}", elem);
    }

    // 排序（需要可变切片 &mut [T]）
    let mut mut_data = data;  // 或 let mut_data = &mut data[..];
    mut_data.sort();
```

## 五、切片与数组、Vec 的自动转换

`Vec<T>` 和数组都可以通过解引用强制转换为切片，因此任何接受 `&[T]` 的函数都可以直接传入 `&Vec<T>` 或数组引用。

```rust
    fn sum(slice: &[i32]) -> i32 {
        slice.iter().sum()
    }

    let arr = [1, 2, 3];
    let v = vec![4, 5, 6];

    sum(&arr);   // &[i32; 3] 转为 &[i32]
    sum(&v);     // &Vec<i32> 转为 &[i32]
```

## 六、不拥有数据的含义

由于切片只是借用，它**不会**管理内存的释放。当被借用的数据离开作用域，切片就会变成悬垂指针（但 Rust 的所有权和生命周期系统在编译时就能阻止这种错误）。

```rust
    // 错误示例：切片比原数据活得久
    // let dangling: &[i32];
    // {
    //     let v = vec![1, 2, 3];
    //     dangling = &v[..];
    // } // v 被释放，dangling 无效 – Rust 编译器会报错
```
