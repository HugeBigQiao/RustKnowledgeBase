# 裸指针

Rust 有两种裸指针类型：`*mut T`（可变裸指针）和 `*const T`（不可变裸指针）。裸指针与 C/C++ 中的指针非常相似，是 Rust 中不受约束的指针类型。必须始终指定 `const` 或 `mut`，没有单纯的 `*T` 类型。

## 特点

- **不受 Rust 借用规则约束**：Rust 不会跟踪裸指针指向的内容
- **可能为空**：裸指针可以为空（null），类似 C 的 `NULL` 或 C++ 的 `nullptr`
- **可能悬空**：可能指向已释放的内存
- **没有自动生命周期管理**
- **只能在 `unsafe` 块中解引用**
- **创建、传递、比较裸指针是安全的**，只有解引用才需要 `unsafe`

## 创建裸指针

### 从引用转换

```rust
let mut x = 10;
let ptr_x = &mut x as *mut i32;       // 可变引用 → 可变裸指针

let y = Box::new(20);
let ptr_y = &*y as *const i32;        // 共享引用 → 不可变裸指针

unsafe {
    *ptr_x += *ptr_y;                 // 只能在 unsafe 中解引用
}
assert_eq!(x, 30);
```

### 从整数构造

```rust
// 可从整数构造裸指针，但通常只有从指针转换来的整数才可信任
let ptr = 0x1234 as *const i32;       // 极度危险，几乎必定未定义行为
```

### 空指针

```rust
let null_ptr: *const i32 = std::ptr::null();
let null_mut: *mut i32 = std::ptr::null_mut();
```

### 从拥有型指针转换

```rust
let b = Box::new(42);
let raw = Box::into_raw(b);           // 消耗 Box，返回裸指针

// 之后必须用 from_raw 回收，否则内存泄漏
unsafe {
    let b = Box::from_raw(raw);       // 重新由 Box 接管
}
```

## 解引用

裸指针 **不会** 被自动解引用：

| 特性 | 裸指针行为 |
|------|------------|
| `.` 运算符 | 不隐式解引用，需 `(*raw).field` |
| `Deref` 特型 | 裸指针没有实现 |
| `==`、`<` 等比较 | 按**地址**比较，非按值 |
| 格式化 `Display` | 不追踪裸指针 |
| 格式化 `Debug` / `Pointer` | 展示十六进制地址 |

```rust
let s = "hello";
let ptr: *const String = &s as *const String;

unsafe {
    // (*ptr).len() 或 ptr.read().len()——都必须显式处理
    let len = (*ptr).len();
}
```

## 指针运算

Rust 的 `+` 运算符不适用于裸指针。使用专用方法：

```rust
let trucks = vec!["garbage truck", "dump truck", "moonstruck"];
let first: *const &str = &trucks[0];
let last: *const &str = &trucks[2];

assert_eq!(unsafe { last.offset_from(first) }, 2);
assert_eq!(unsafe { first.offset_from(last) }, -2);
```

| 方法 | 说明 |
|------|------|
| `offset(n)` | 偏移 n 个元素，不得越界 |
| `wrapping_offset(n)` | 回绕型偏移，允许越界 |
| `add(n)` / `sub(n)` | 更便捷的 offset 封装 |
| `wrapping_add(n)` / `wrapping_sub(n)` | 回绕型加减 |
| `offset_from(other)` | 计算两个指针之间的元素距离 |

## 可空指针转换

```rust
fn option_to_raw<T>(opt: Option<&T>) -> *const T {
    match opt {
        None => std::ptr::null(),
        Some(r) => r as *const T,
    }
}

assert!(!option_to_raw(Some(&("pea", "pod"))).is_null());
assert_eq!(option_to_raw::<i32>(None), std::ptr::null());
```

`as_ref` 和 `as_mut` 方法安全地转换回引用：

```rust
// *const T → Option<&T>
let opt_ref = (ptr as *const i32).as_ref();

// *mut T → Option<&mut T>
let opt_mut = (ptr as *mut i32).as_mut();
```

## 胖指针

指向动态大小类型（DST）的裸指针也是胖指针：

- `*const [u8]` 包含长度 + 地址
- `*mut dyn std::io::Write` 包含虚表指针 + 数据指针

```rust
let slice: &[i32] = &[1, 3, 9];
let raw: *const [i32] = slice as *const [i32];  // 胖裸指针
```

## 安全解引用指南

1. **不可解引用空指针或悬空指针**（未定义行为）
2. **不可解引用未正确对齐的指针**（未定义行为）
3. **借用时必须遵守引用规则**：共享访问只读，可变访问独占
4. **必须确保引用目标是格式良好的值**（例如 `*const char` 必须是合法 Unicode 码点）
5. **`offset` 只能偏移到同一变量/分配区域内部**，或紧邻其后的第一个字节
6. **写入时不得破坏类型不变条件**（例如 `*mut u8` 指向 `String` 内部时，写入必须保持 UTF-8 合法性）

## Send 与 Sync

裸指针**既不是 `Send` 也不是 `Sync`**，因此包含裸指针的类型默认不会自动实现这些特型。如果确实需要跨线程传递裸指针，需手动实现 `Send` / `Sync`（见不安全代码模块），但这属于高级用法。

## 与引用的关键对比

| 特性 | 引用 `&T` / `&mut T` | 裸指针 `*const T` / `*mut T` |
|------|----------------------|------------------------------|
| 编译期安全检查 | 是 | 否 |
| 解引用 | 安全，在普通代码中 | 需 `unsafe` 块 |
| 可为空 | 否 | 是 |
| 可悬空 | 不允许 | 允许但危险 |
| 自动解引用 | 是（点运算符） | 否 |
| 线程安全标记 | 自动实现 Send/Sync | 不实现 |
| 典型用途 | 日常安全编程 | FFI、自定义数据结构、位级 hack |

## 典型应用场景

1. **FFI（外部函数接口）**：与 C/C++ 代码交互
2. **自定义数据结构**：双向链表、任意对象图等 Rust 借用检查器不允许的结构
3. **位级优化**：如 `RefWithFlag`——在一个机器字中同时存储引用和布尔标志
4. **避免借用检查器限制**：某些合法但借用检查器无法验证的模式

## 示例：RefWithFlag

利用对齐约束的最低有效位存储额外信息的经典技术：

```rust
mod ref_with_flag {
    use std::marker::PhantomData;
    use std::mem::align_of;

    pub struct RefWithFlag<'a, T> {
        ptr_and_bit: usize,
        behaves_like: PhantomData<&'a T>,   // 不占空间，帮助生命周期推断
    }

    impl<'a, T: 'a> RefWithFlag<'a, T> {
        pub fn new(ptr: &'a T, flag: bool) -> RefWithFlag<T> {
            assert!(align_of::<T>() % 2 == 0);  // 对齐必须为偶数
            RefWithFlag {
                ptr_and_bit: ptr as *const T as usize | flag as usize,
                behaves_like: PhantomData,
            }
        }

        pub fn get_ref(&self) -> &'a T {
            unsafe {
                let ptr = (self.ptr_and_bit & !1) as *const T;
                &*ptr
            }
        }

        pub fn get_flag(&self) -> bool {
            self.ptr_and_bit & 1 != 0
        }
    }
}
```

此例在一个 `usize` 中同时存储了引用和 `bool`，比元组 `(&T, bool)` 节省了一半空间。`PhantomData<&'a T>` 告知 Rust 此类型应被视为包含一个引用，以确保正确的生命周期推断。
