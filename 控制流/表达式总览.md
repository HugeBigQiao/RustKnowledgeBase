# Rust 表达式总览

表达式是 Rust 语言最底层的求值哲学。**在 Rust 中，几乎一切都是表达式，都会产生一个值。** 理解这一点，是读懂 Rust 代码美感与简洁性的关键。

## 一、语句与表达式的根本区别

Rust 中只有两种语法单元：

- **语句**：执行操作，**不返回值**。以分号 `;` 结尾。
- **表达式**：计算并**产出一个值**。没有分号结尾。

```rust
    // 语句：let 绑定、函数定义、模块声明等
    let x = 5;         // 语句，不返回值

    // 表达式：字面量、运算、块、if、loop 等
    5 + 3              // 表达式，值为 8
    x + 1              // 表达式，值为 6
```

这个区别看似简单，但深刻影响 Rust 代码的书写方式。

## 二、表达式无处不在：一个全景视图

以下语法结构在 Rust 中**全部都是表达式**，都会产生一个值：

- 字面量：`42`、`"hello"`、`true`
- 变量名：`x`
- 运算：`a + b`、`x > 0`、`!flag`
- 块：`{ ... }`
- `if` 表达式
- `match` 表达式
- `loop` 表达式（`while` 和 `for` 不是，它们固定返回 `()`）
- 闭包：`|x| x + 1`
- `return` 表达式
- `break` 表达式（带值）
- 函数调用：`foo()`
- 结构体/枚举构造：`Point { x: 1, y: 2 }`
- 范围：`0..10`、`..5`

## 三、块表达式的中心地位

块 `{ }` 是一个表达式，它的值是**最后一个无分号表达式的值**。这使得多步骤计算可以作为一个值存在。

```rust
    let y = {
        let x = 3;
        x + 1    // 没有分号，这是块的值
    };
    // y == 4
```

如果块的最后一行加了分号，它就变成了语句，块的返回值变为单元类型 `()`。

```rust
    let y = {
        let x = 3;
        x + 1;   // 分号让这一行变成语句
    };
    // y 的类型是 ()，值是空元组
```

理解块表达式的这个行为，是理解 `if` 和 `match` 如何返回值的基础。

## 四、分号的语义：抑制值

分号在 Rust 中有明确的语义：**将一个表达式变为语句，并丢弃其产生的值**。表达式 `x + 1` 产生一个值，但 `x + 1;` 就变成了语句，不产生有意义的返回值（它的结果类型是 `()`）。

这种设计允许编译器在类型层面区分"有值"和"无值"的上下文。

```rust
    // 函数体是表达式
    fn add_one(x: i32) -> i32 {
        x + 1  // 没有分号，表达式的值就是函数的返回值
    }

    // 有分号的话就不会返回该值
    fn do_something(x: i32) {
        x + 1; // 语句，返回值被丢弃，函数返回 ()
    }
```

## 五、哪些语法结构不是表达式？

并非 Rust 中的一切语法都是表达式。以下是典型的**非表达式**结构：

- `let` 语句
- `fn` 函数定义
- `mod` 模块声明
- `use` 导入声明
- `struct` / `enum` / `trait` 定义
- `impl` 块本身（但其中的函数是表达式）
- `while` 循环（返回 `()`）
- `for` 循环（返回 `()`）

这些结构被称为**项**或**语句**，它们不产生有意义的运行时值，而是组织代码的骨架。

## 六、表达式的类型与一致性

表达式的类型在编译时确定，且要求所有可能的分支返回值类型一致。

- `if` 表达式的每个分支必须返回相同类型。
- `match` 表达式的每个 arm 必须返回相同类型。
- 块的最终表达式类型决定了整个块的类型。

```rust
    let n = 5;
    let result = if n > 0 {
        1         // i32
    } else {
        -1        // i32，类型一致
    };

    // 错误示例：分支类型不一致
    // let x = if true { 42 } else { "hello" };
```

## 七、表达式导向的设计哲学

Rust 以表达式为核心的设计有几个明确优点：

- **消除空值/未初始化变量**：变量总是由表达式初始化，不存在"声明了但没赋值"的状态。
- **函数式风格的流畅表达**：`let x = if cond { a } else { b };` 这类写法简洁且不可变。
- **类型安全**：编译时就能检测分支返回值类型是否一致。
- **与所有权系统的协同**：表达式产出值，值的所有权生命周期清晰可追踪。

## 速记要点

- 语句执行操作不返回值，表达式计算并产出一个值
- 块 `{}` 是表达式，值是最后一个无分号表达式的值
- 分号将表达式变为语句，丢弃其值并返回 `()`
- `if`、`match`、`loop` 都是表达式，可以返回值
- `while`、`for`、`let` 不是表达式，返回 `()`
- 表达式让 Rust 代码简洁、安全且富有表达力
