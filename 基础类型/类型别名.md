# Rust 类型别名

类型别名使用 `type` 关键字为现有类型定义一个别名。它不会创建新类型，只是给已有类型取一个更简短或更语义化的名字。

## 一、基本语法

```rust
// 基本形式
type Name = ExistingType;

// 常见用法：简化长类型名
type Kilometers = i32;
type Thunk = Box<dyn Fn() + Send + 'static>;
```

## 二、主要用途

### 2.1 简化长类型名

```rust
// 不用别名时：类型签名冗长
fn take_writer(w: Box<dyn Write + Send + 'static>) { /* ... */ }
fn return_writer() -> Box<dyn Write + Send + 'static> { /* ... */ }

// 使用别名后：清晰简洁
type GenericWriter = Box<dyn Write + Send + 'static>;

fn take_writer(w: GenericWriter) { /* ... */ }
fn return_writer() -> GenericWriter { /* ... */ }
```

### 2.2 为泛型参数添加语义

```rust
// 虽然都是 i32，但语义完全不同
type Meters = i32;
type Seconds = i32;

let distance: Meters = 100;
let time: Seconds = 30;
```

### 2.3 泛型类型别名

```rust
// 泛型类型别名可以减少泛型参数的重复
type Result<T> = std::result::Result<T, std::io::Error>;
// 此后只需写 Result<T>，不必每次写 Result<T, io::Error>
```

## 三、注意事项

- **类型别名不是新类型**：`type Kilometers = i32;` 之后，`Kilometers` 和 `i32` 是完全可互换的同一类型。如果需要区分不同类型，应使用**新型模式**（newtype pattern）。

```rust
// 新型模式：真正的独立类型
struct Kilometers(i32);

let k = Kilometers(42);
let n: i32 = 42;
// k == n;  // 编译错误：不同类型不能直接比较
```

- **类型别名与 C 的 typedef 类似**，但 Rust 的 `type` 不支持为泛型参数指定部分类型参数（需要使用 trait alias 或其他方式）。
