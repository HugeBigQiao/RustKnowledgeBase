## 四种溢出处理方法

### 1. `wrapping_*` – 回绕

直接按二进制补码规则回绕。溢出时从最小值重新开始，下溢时从最大值重新开始。

- **返回值**：同类型
- **适用场景**：哈希、加密、模运算等明确需要循环行为的场景

**示例**：

``` rust
let a: u8 = 255;
let b = a.wrapping_add(1);   // 0
let c = a.wrapping_add(2);   // 1

let d: u8 = 0;
let e = d.wrapping_sub(1);   // 255
```

### 2. `checked_*` – 检查溢出

检查溢出。溢出时返回 `None`，否则返回 Some(结果)。

- **返回值**：`Option<T>`
- **适用场景**：需要明确处理异常情况，不能接受任何静默错误的业务逻辑

**示例**：

``` rust
let a: u8 = 255;
let b = a.checked_add(1);    // None

let c: u8 = 100;
let d = c.checked_add(50);   // Some(150)
```

### 3. `overflowing_*` – 溢出标志

返回一个元组 `(结果, 是否溢出)`。结果始终是回绕后的值，第二个布尔值告诉你是否发生了溢出。

- **返回值**：`(T, bool)`
- **适用场景**：既想知道回绕后的值，又要记录是否溢出，便于后续决策或日志

**示例**：

``` rust
let a: u8 = 255;
let (result, overflowed) = a.overflowing_add(1);
// result = 0, overflowed = true

let b: u8 = 100;
let (result, overflowed) = b.overflowing_add(50);
// result = 150, overflowed = false
```

### 4. `saturating_*` – 饱和运算

溢出时不会回绕，而是“饱和”到该类型能表示的最大值或最小值。

- **返回值**：同类型
- **适用场景**：音量、亮度等有物理上限的值，不希望从最大值跳回零。

**示例**：

``` rust
let a: u8 = 250;
let b = a.saturating_add(20);   // 255，而不是 14

let c: u8 = 5;
let d = c.saturating_sub(10);   // 0
```
