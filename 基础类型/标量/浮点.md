## Rust 浮点类型

    Rust 提供了两种原生浮点类型 `f32` 和 `f64`，遵循 IEEE 754 标准。它们用于表示带小数的数字。

## 一、速览对比

| 类型 | 位宽 | 精度 | 默认值 | 默认类型 | 常用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `f32` | 32 | 单精度 (约7位小数) | `0.0` | 否 | 图形、嵌入式、对内存敏感的场景 |
| `f64` | 64 | 双精度 (约15位小数) | `0.0` | **是** | **默认浮点类型**，科学与工程计算 |

## 二、IEEE 754 标准与特殊值

浮点数与整数最大的不同在于：它有一些合法的“特殊值”，这属于设计标准，而不是错误。

- **正负无穷 (`∞`)**：由 `1.0 / 0.0` 等操作产生，它代表着数学上的“无穷大”概念。

    ```rust
    let positive_infinity = f64::INFINITY;
    let negative_infinity = f64::NEG_INFINITY;
    ```

- **NaN**（Not a Number）：表示非数字值，由 `0.0 / 0.0` 等操作产生。
    - 关键特性：NaN 不等于任何值，包括它自己。

    ```rust
    let nan = f64::NAN;
    // 判断 NaN 必须用 is_nan()，不能用 ==
    assert!(nan.is_nan());
    ```

## 三、处理方法

因为浮点数溢出（变为 ∞）或无效运算（变为 NaN）不会导致 panic，所以处理方法主要是状态检查，最常用的有：

- `is_infinite()`：检查是否为无穷大或无穷小。
  - 返回值：`bool`
  - 适用场景：需要判断浮点数是否为无穷大或无穷小，不能接受任何静默错误的业务逻辑。
  - **示例**：

  ```rust
  let a = f64::INFINITY;
  assert!(a.is_infinite());
  ```

- `is_nan()`：检查是否为 NaN。
  - 返回值：`bool`
  - 适用场景：需要判断浮点数是否为 NaN，不能接受任何静默错误的业务逻辑。
  - **示例**：

  ```rust
  let a = f64::NAN;
  assert!(a.is_nan());
  ```

- `is_finite()`：检查是否为有限值。
  - 返回值：`bool`
  - 适用场景：需要判断浮点数是否为有限值，不能接受任何静默错误的业务逻辑。
  - **示例**：

    ```rust
    let a = 1.0;
    assert!(a.is_finite());
    ```

**示例**：

    ```rust
    let num: f64 = 1.0 / 0.0;
    assert!(num.is_infinite());
    assert!(!num.is_finite());
    ```

### 四、主要特性与陷阱

- **精度问题**：浮点数在二进制中无法精确表示所有十进制小数，这会导致经典问题：

    ```rust
    // 经典的精度损失
    let sum = 0.1 + 0.2;
    // sum 不等于 0.3，而是 0.30000000000000004
    ```

- **没有Eq实现**：NaN != NaN，浮点数无法满足“等于自身”这一数学公理。因此 f32 和 f64 没有实现 std::cmp::Eq trait。这意味着你不能直接用 == 比较两个浮点数是否完全相等，更推荐检查它们的差的绝对值是否小于一个极小值（epsilon）。

- **安全转换**：从整数转为浮点通常是安全的（可能丢失精度），但从浮点转回整数可能遇到无穷、NaN 或超出范围。一般建议使用 as 转换符进行安全转换。

    ```rust
    let num: f64 = 1.0;
    let int: i32 = num as i32;
    assert_eq!(int, 1);
    ```
