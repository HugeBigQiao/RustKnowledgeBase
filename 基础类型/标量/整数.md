## 基础概念

Rust 提供固定宽度的整数类型，宽度从 8 位到 128 位，分为有符号（ i ）和⽆符号（ u ）。
Rust 的有符号整型会使⽤⼆进制补码表⽰，使⽤与相应的⽆符号类型相同的位模式来覆盖正值和负值的范围

### 为什么需要不同宽度？

- 内存效率：存储年龄⽤ u8 （1 字节）就够了，⽤ u64 （8 字节）是浪费
- 硬件亲和：CPU 的寄存器和 ALU 都是围绕固定位宽设计的
- 语义明确：⽆符号类型适合年龄、索引等永不为负的数据

## 一、速览对比

### 有符号整数(可表示负数)

| 类型 | 位宽 | 取值范围 | 常用场景 |
| ------ | ------ | ---------- | ---------- |
| `i8` | 8 | -128 ~ 127 | 小整数、字节级运算（需负数） |
| `i16` | 16 | -32,768 ~ 32,767 | 较少用 |
| `i32` | 32 | -21亿 ~ 21亿 | **默认整数类型**，性能与范围平衡 |
| `i64` | 64 | -9e18 ~ 9e18 | 大数、时间戳 |
| `i128` | 128 | 巨大 | 极少数需要超大数的场景 |
| `isize` | 架构决定(32/64) | 同 `i32` 或 `i64` | 容器索引、与指针打交道 |

### 无符号整数（非负）

| 类型 | 位宽 | 取值范围 | 常用场景 |
| ------ | ------ | ---------- | ---------- |
| `u8` | 8 | 0 ~ 255 | **字节操作**，如 `&[u8]` 读文件 |
| `u16` | 16 | 0 ~ 65,535 | 少见 |
| `u32` | 32 | 0 ~ 42亿 | 正索引、位掩码 |
| `u64` | 64 | 0 ~ 1.8e19 | 大容量统计 |
| `u128` | 128 | 0 ~ 3.4e38 | 哈希、极端场景 |
| `usize` | 架构决定(32/64) | 同 `u32` 或 `u64` | **容器长度、索引的默认类型** |

## 二、逐类型详解

### i8 / u8

- **位宽**：8 位
- **字节数**：1
- **有符号(i8)**：-128 ~ 127
- **无符号(u8)**：0 ~ 255
- **默认值**：0
- **特殊性**：
  - `u8` 是字节操作的核心，所有文件 I/O 的原始缓冲都是 `&[u8]`。
  - `i8` 有时在 FFI 或处理 C 接口时出现。
- **示例**：

    ``` rust
    let a: i8 = -42; 
    let b: u8 = 255;
    ```

### i16 / u16

- **位宽**：16 位
- **字节数**：2
- **有符号(i16)**：-32,768 ~ 32,767
- **无符号(u16)**：0 ~ 65,535
- **默认值**：0
- **特殊性**：  
  - 日常极少使用，主要在嵌入式或特定协议解析中遇到。
- **示例**：
  
    ``` rust
    let x: i16 = 20_000;
    let y: u16 = 60000;
    ```

### i32 / u32

- **位宽**：32 位
- **字节数**：4
- **有符号(i32)**：-2,147,483,648 ~ 2,147,483,647(-21亿 - 21亿)
- **无符号(u32)**：0 ~ 4,294,967,295(0 - 42亿)
- **默认值**：0
- **特殊性**：  
  - i32 是 Rust 整数字面量的默认类型。写 let x = 42; 时，x 就是 i32。
  - 性能通常是所有整数里最优的。
- **示例**：
  
    ``` rust
    let default_int = 42; // 自动推断为 i32
    let mask: u32 = 0xFFFF_0000;
    ```

### i64 / u64

- **位宽**：64位
- **字节数**：8
- **取值范围**：非常广，在目前的 Rust 中，i64 可以表示 -9e18 ~ 9e18，u64 可以表示 0 ~ 1.8e19。
- **默认值**：0
- **特殊性**：处理大整数、时间戳、统计量的主力。
- **示例**：
  
    ``` rust
    let big: i64 = 9_223_372_036_854_775_807;
    let ts: u64 = 1716819200; // Unix 时间戳
    ```

### i128 / u128

- **位宽**：128 位
- **字节数**：16
- **取值范围**：天文数字。
- **默认值**：0
- **特殊性**：极少用，但在加密、大数运算或追求绝对不溢出的安全场景会遇到。注意运算开销较大。
- **示例**：
  
    ``` rust
    let huge: u128 = 340_282_366_920_938_463_463_374_607_431_768_211_455;
    ```

### isize / usize

- **位宽**：根据 CPU 架构决定（32 位系统为 32 位，64 位系统为 64 位）
- **字节数**：4 或 8
- **有符号(isize)**：与指针等宽的有符号整数
- **无符号(usize)**：与指针等宽的无符号整数
- **默认值**：0
- **特殊性**：  
  - usize 是索引和长度的标准类型。数组 arr[i] 中的 i 必须是 usize。
  - isize 较少用，但当你需要指针差值可能为负时会用到。
- **示例**：

    ``` rust
    let arr = [10, 20, 30];
    let idx: usize = 1;
    println!("{}", arr[idx]); // 20

    let len = arr.len(); // len 的类型是 usize
    ```

## 三、整数溢出处理

在 Rust 中，整数都有一个固定的位宽（如 `u8` 只有 8 位）。当运算结果超过该类型能表示的范围时，就叫**溢出**。

- **Debug 模式**：溢出会立即触发 `panic`，程序崩溃，强制你发现错误。
- **Release 模式**：为了性能，溢出会**静默回绕**，比如 `255u8 + 1` 变成 `0`，这可能引入隐蔽的 bug。
