# Rust 字符串类型

Rust 的字符串分为两种：拥有所有权的 `String` 和借用的 `&str`。理解它们的关系与区别，是掌握所有权和内存管理的关键一步。

## 一、速览对比

|  | `String` | `&str` |
| :--- | :--- | :--- |
| 本质 | 拥有所有权的、可增长的 UTF-8 字节缓冲区 | 借用的、不可变的字符串切片 |
| 存储位置 | 堆 | 指向别处（栈、堆、静态区）的胖指针 |
| 可变性 | 可以修改、追加、删除 | 只读，不可修改 |
| 拥有数据 | 是 | 否，只是借用 |
| 类比 | 一整个完整的信封 | 信封上的地址贴纸 |
| 默认值 | 无（通过 `String::new()` 创建空串） | 无（通常作为引用参数传入） |

## 二、`String` – 可变的堆上字符串

`String` 是 Rust 中最常用的字符串类型，它拥有数据的所有权。当变量离开作用域时，堆上分配的字符串数据会被自动释放。

```rust
    //创建 `String`
    // 从字符串字面量创建
    let s1 = String::from("hello");

    // 使用 to_string()
    let s2 = "hello".to_string();

    // 创建一个空字符串
    let mut s3 = String::new();

    //修改 `String`
    // `String` 支持多种修改操作，这些操作可能触发堆上的重新分配。
    let mut s = String::from("hello");

    // 追加字符串切片
    s.push_str(" world");

    // 追加单个字符
    s.push('!');

    // 使用 + 运算符（会转移左值所有权）
    let s1 = String::from("hello");
    let s2 = String::from(" world");
    let s3 = s1 + &s2;  // s1 的所有权被移动到 s3，s1 不再可用

    // 使用 format! 宏（不转移所有权）
    let s1 = String::from("hello");
    let s2 = String::from("world");
    let s3 = format!("{} {}", s1, s2); // s1 和 s2 仍然可用
```

### `String` 的内存布局

`String` 在栈上保存了三个字段：
    - 指向堆上字节数组的指针
    - 当前已使用的长度
    - 容量（已分配的堆空间大小）

当 `push_str` 导致长度超过容量时，`String` 会自动在堆上重新分配一块更大的空间。

## 三、`&str` – 不可变的字符串切片

`&str` 是一个借用，不拥有数据。它是对某段 UTF-8 字节序列的一个“视图”。几乎所有接受字符串参数的函数都使用 `&str`，以便同时接受 `String` 和字符串字面量。

### `&str` 的几种来源

```rust
    // 字符串字面量本身就是 &str，其数据存储在二进制文件的静态区
    let literal: &str = "hello";

    // 从 String 借用
    let owned = String::from("hello");
    let borrowed: &str = &owned;

    // 从字符串切片获取
    let sliced: &str = &owned[0..2];  // "he"
```

### `&str` 是胖指针

一个 `&str` 在内存中占用两个 `usize` 大小：一个指向数据起始地址的指针，以及一个长度。这使得切片操作极其高效——它只是调整指针和长度，不复制数据。

## 四、相互转换`String` → `&str`

### `String` → `&str`

这种转换非常廉价，因为 Rust 会自动解引用：

```rust
    let owned = String::from("hello");
    let borrowed: &str = &owned;  // 显式借用

    fn greet(name: &str) {
        println!("Hello, {}!", name);
    }
    greet(&owned);  // 自动解引用，传入 &str
```

### `&str` → `String`

需要显式地创建新的堆上数据：

```rust
    let borrowed = "hello";

    let owned1 = borrowed.to_string();
    let owned2 = String::from(borrowed);
    let owned3: String = borrowed.into();
```

## 五、索引与遍历

**Rust 字符串不能直接用索引访问**。因为 UTF-8 是变长编码，`s[0]` 不一定能拿到完整的第一个字符。

```rust
    let s = String::from("你好");

    // 遍历字节
    for b in s.bytes() {
        println!("{}", b);  // 依次打印各字节的值
    }

    // 遍历字符
    for c in s.chars() {
        println!("{}", c);  // 依次打印 '你'、'好'
    }
```

如果需要按字节索引切片，必须确保切在合法的字符边界上，否则会 panic。

## 六、其他实用方法

`String` 和 `&str` 共享大量字符串操作方法：

```rust
    let s = String::from("hello world");

    s.len();                    // 字节长度（11）
    s.is_empty();               // 是否为空
    s.contains("world");        // 是否包含子串
    s.starts_with("he");        // 是否以某前缀开头
    s.ends_with("ld");          // 是否以某后缀结尾
    s.replace("world", "Rust"); // 替换所有匹配
    s.trim();                   // 去除首尾空白
    s.split_whitespace();       // 按空白分割，返回迭代器
```

## 七、`String` 与 `Vec<u8>` 的关系

`String` 本质上是 `Vec<u8>` 的一个包装，但额外保证内容是合法的 UTF-8 编码。如果需要直接操作字节，可以在这两者之间转换：

```rust
    // String → Vec<u8>
    let s = String::from("hello");
    let bytes: Vec<u8> = s.into_bytes();

    // Vec<u8> → String（需要保证是合法 UTF-8）
    let bytes = vec![104, 101, 108, 108, 111]; // "hello"
    let s = String::from_utf8(bytes).unwrap();
```
