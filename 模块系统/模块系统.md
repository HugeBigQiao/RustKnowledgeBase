# Rust 模块系统

Rust 使用**模块**（module）在项目内部组织代码。模块是函数、类型、常量等的命名空间容器，配合可见性控制和路径系统构成 Rust 的代码组织骨架。

> 项目构建、依赖管理、测试、发布等内容请参见：[Cargo与项目管理](Cargo与项目管理.md)。

---

## 一、模块基础

模块是函数、类型、常量等的容器，充当 Rust 的命名空间。

### 1.1 基本定义

```rust
mod spores {
    use cells::{Cell, Gene};

    pub struct Spore { ... }          // 公开
    pub fn produce_spore(...) { ... } // 公开
    pub(crate) fn genes(...) { ... }  // crate 内公开
    fn recombine(...) { ... }         // 私有（默认）
}
```

| 可见性修饰符 | 含义 |
| :--- | :--- |
| （无） | 私有：仅当前模块及子模块可见 |
| `pub` | 公开：外部可访问 |
| `pub(crate)` | crate 内公开，不作为外部接口 |
| `pub(super)` | 仅父模块可见 |
| `pub(in path)` | 在指定路径的模块及其后代中可见 |

### 1.2 嵌套模块

```rust
mod plant_structures {
    pub mod roots { ... }
    pub mod stems { ... }
    pub mod leaves { ... }
}
```

> 如果希望嵌套模块中的项对外部 crate 可见，必须将**该项及其所在的模块**都标记为 `pub`。

### 1.3 模块与文件系统

Rust 支持 3 种模块组织方式：

```
fern_sim/
├── Cargo.toml
└── src/
    ├── main.rs             # 声明: pub mod plant_structures;
    ├── spores.rs           # 独立模块文件
    └── plant_structures/
        ├── mod.rs           # 声明: pub mod roots; pub mod stems; pub mod leaves;
        ├── leaves.rs
        ├── roots.rs
        └── stems/           # 带子模块的目录+同名文件
            ├── phloem.rs
            ├── xylem.rs
            └── stems.rs     # 声明子模块: pub mod xylem; pub mod phloem;
```

规则：
- `mod spores;` → 查找 `spores.rs` 或 `spores/mod.rs`
- 目录下的 `mod.rs` 代表该模块本身
- 可以同时使用 `stems.rs` + `stems/` 目录存放子模块

## 二、路径与导入

```rust
// 绝对路径
std::mem::swap(&mut s1, &mut s2);

// use 导入
use std::mem;
mem::swap(&mut s1, &mut s2);

// 批量导入
use std::collections::{HashMap, HashSet};
use std::fs::{self, File};   // 同时导入 fs 模块和 File
use std::io::prelude::*;     // 导入所有公开项

// 重命名
use std::io::Result as IOResult;

// 路径前缀
use super::AminoAcid;        // 父模块
use crate::proteins::AminoAcid;  // crate 根
use self::image::Sampler;    // 当前模块下的子模块

// 消歧义：以 :: 开头的绝对路径永远引用外部 crate
use ::image::Pixels;         // image crate
use self::image::Sampler;    // 当前模块中的 image 模块
```

> 每个模块以"白板"开始，不会自动继承父模块的名称。必须显式导入需要的内容。

### 标准库预导入

每个模块（包括根模块）都隐式包含：

```rust
use std::prelude::v1::*;
```

这会自动导入常用特型和类型（如 `Vec`、`Result`、`Option`、`String`、`Box`、`Clone`、`Copy`、`Drop`、`Iterator` 等）。

### 公开 `use` 声明

```rust
// 在 plant_structures/mod.rs 中
pub use self::leaves::Leaf;
pub use self::roots::Root;
```

这使得 `Leaf` 和 `Root` 成为 `plant_structures` 模块的直接成员，为标准库预导入的编写方式。

### 公开结构体字段

```rust
pub struct Fern {
    pub roots: RootSet,  // 公开字段
    pub stems: StemSet   // 公开字段
}
```

- 在声明结构体的模块及其子模块中，所有字段（包括私有字段）均可访问
- 模块外只能访问 `pub` 字段
- 通过**模块**而非类来实施访问控制，减少了 getter/setter 样板代码

### 静态变量与常量

```rust
// 常量（编译时内联，类似 C++ #define）
pub const ROOM_TEMPERATURE: f64 = 20.0;  // 命名规约：UPPER_CASE

// 静态变量（程序启动前初始化，持续到退出）
pub static ROOM_TEMPERATURE: f64 = 68.0;
```

| | `const` | `static` |
| :--- | :--- | :--- |
| 值内联到使用处 | 是 | 否（固定内存地址） |
| 可取引用 | 否 | 是 |
| 可变 | 不可 | `static mut`（非线程安全，安全代码不可用） |

> 对魔数和字符串使用常量，对大量数据或需借用引用时使用静态变量。

## 三、属性

属性是 Rust 的通用语法，用于向编译器提供指令和建议。

```rust
#[allow(non_camel_case_types)]     // 禁用警告
#[cfg(target_os = "android")]      // 条件编译
#[inline]                          // 建议内联
#[inline(always)] / #[inline(never)]  // 强制内联/不内联
```

**常用 `#[cfg]` 选项：**

| 条件 | 启用时机 |
| :--- | :--- |
| `test` | `cargo test` 或 `rustc --test` |
| `debug_assertions` | 调试构建（非优化） |
| `unix` / `windows` | 对应平台 |
| `target_pointer_width = "64"` | 64 位平台 |
| `target_os = "macos"` / `"linux"` / ... | 特定操作系统 |
| `feature = "robots"` | 启用自定义特性 |
| `not(A)` / `all(A, B)` / `any(A, B)` | 逻辑组合 |

**crate 级属性**使用 `#!` 语法：

```rust
// lib.rs 顶部
#![allow(non_camel_case_types)]
#![feature(trace_macros)]  // 启用不稳定特性
```

## 四、程序变库

将程序改造成库的步骤：

1. 将 `src/main.rs` 重命名为 `src/lib.rs`
2. 给公开项添加 `pub`
3. 将 `main` 函数移到 `src/bin/` 目录中

```
fern_sim/
├── Cargo.toml
└── src/
    ├── lib.rs             # 库的根模块
    └── bin/
        ├── efern.rs       # use fern_sim::{Fern, run_simulation};
        └── draw_fern/     # 大型程序可用子目录
            ├── main.rs
            └── draw.rs
```

Cargo 的默认行为：
- `src/lib.rs` 存在 → 构建库
- `src/main.rs` 存在 → 构建可执行文件
- `src/bin/*.rs` → 构建额外的可执行程序
