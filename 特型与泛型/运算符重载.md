# 运算符重载

在 Rust 中，像 `a + b` 这样的表达式实际上是 `a.add(b)` 的简写——对标准库中特定特型方法的调用。只要为自定义类型实现这些内置特型，就能支持相应的运算符。这称为**运算符重载**。

## 一、运算符重载特型汇总表

| 类别 | 特型 | 运算符 |
|------|------|--------|
| 一元运算符 | `std::ops::Neg`, `std::ops::Not` | `-x`, `!x` |
| 算术运算符 | `std::ops::Add`, `Sub`, `Mul`, `Div`, `Rem` | `x + y`, `-`, `*`, `/`, `%` |
| 按位运算符 | `std::ops::BitAnd`, `BitOr`, `BitXor`, `Shl`, `Shr` | `x & y`, `\|`, `^`, `<<`, `>>` |
| 复合赋值（算术） | `std::ops::AddAssign`, `SubAssign`, `MulAssign`, `DivAssign`, `RemAssign` | `x += y`, `-=`, `*=`, `/=`, `%=` |
| 复合赋值（按位） | `std::ops::BitAndAssign`, `BitOrAssign`, `BitXorAssign`, `ShlAssign`, `ShrAssign` | `x &= y`, `\|=`, `^=`, `<<=`, `>>=` |
| 相等性比较 | `std::cmp::PartialEq`, `std::cmp::Eq` | `x == y`, `x != y` |
| 有序比较 | `std::cmp::PartialOrd`, `std::cmp::Ord` | `x < y`, `x <= y`, `x > y`, `x >= y` |
| 索引 | `std::ops::Index`, `std::ops::IndexMut` | `x[y]`, `&x[y]` / `x[y] = z`, `&mut x[y]` |

## 二、算术运算符与按位运算符

### 2.1 Add 特型定义

```rust
trait Add<Rhs = Self> {
    type Output;
    fn add(self, rhs: Rhs) -> Self::Output;
}
```

- **Rhs** 类型参数默认为 `Self`，表示右操作数类型
- **Output** 关联类型描述加法结果的类型
- 如果希望将 `i32` 和 `u32` 同时加到自定义类型上，必须分别实现 `Add<i32>` 和 `Add<u32>`

### 2.2 Complex<T> 的泛型实现

```rust
use std::ops::Add;

impl<T> Add for Complex<T>
where
    T: Add<Output = T>,
{
    type Output = Self;
    fn add(self, rhs: Self) -> Self {
        Complex {
            re: self.re + rhs.re,
            im: self.im + rhs.im,
        }
    }
}
```

通过 `where T: Add<Output = T>` 限定，确保组件类型本身支持加法并返回同类型。

也可以写成更泛化的形式（左右操作数类型独立）：

```rust
impl<L, R> Add<Complex<R>> for Complex<L>
where
    L: Add<R>,
{
    type Output = Complex<L::Output>;
    fn add(self, rhs: Complex<R>) -> Self::Output {
        Complex {
            re: self.re + rhs.re,
            im: self.im + rhs.im,
        }
    }
}
```

但实际中 Rust 倾向于避免混合类型运算，上述泛化版本用处有限。

### 2.3 一元运算符

| 特型名称 | 表达式 | 等效表达式 |
|----------|--------|-----------|
| `std::ops::Neg` | `-x` | `x.neg()` |
| `std::ops::Not` | `!x` | `x.not()` |

```rust
trait Neg {
    type Output;
    fn neg(self) -> Self::Output;
}

trait Not {
    type Output;
    fn not(self) -> Self::Output;
}
```

- Rust 所有带符号数值类型实现了 `Neg`；整数类型和 `bool` 实现了 `Not`
- `!` 运算符对 `bool` 进行逻辑取反，对整数执行按位取反（同时扮演 C/C++ 中 `!` 和 `~` 的角色）

Neg 的 Complex 示例：

```rust
use std::ops::Neg;

impl<T> Neg for Complex<T>
where
    T: Neg<Output = T>,
{
    type Output = Complex<T>;
    fn neg(self) -> Complex<T> {
        Complex {
            re: -self.re,
            im: -self.im,
        }
    }
}
```

### 2.4 二元运算符对应表

| 类别 | 特型名称 | 表达式 | 等效表达式 |
|------|----------|--------|-----------|
| 算术 | `Add`, `Sub`, `Mul`, `Div`, `Rem` | `x + y` 等 | `x.add(y)` 等 |
| 按位 | `BitAnd`, `BitOr`, `BitXor`, `Shl`, `Shr` | `x & y` 等 | `x.bitand(y)` 等 |

所有二元运算符特型都有统一的形式（以 `BitXor` 为例）：

```rust
trait BitXor<Rhs = Self> {
    type Output;
    fn bitxor(self, rhs: Rhs) -> Self::Output;
}
```

Rust 所有数值类型实现了算术运算符；整数类型和 `bool` 实现了按位运算符。

### 2.5 String 的 `+` 运算符

可以使用 `+` 将 `String` 与 `&str` 或另一个 `String` 连接，但左边必须是 `String`（不允许 `&str + …`），以避免通过重复连接左侧小片段构建长字符串造成 O(n²) 性能问题。一般建议使用 `write!` 宏从小片段构建字符串。

## 三、复合赋值运算符

`x += y` 是方法调用 `x.add_assign(y)` 的简写，其中 `add_assign` 是 `AddAssign` 特型的方法：

```rust
trait AddAssign<Rhs = Self> {
    fn add_assign(&mut self, rhs: Rhs);
}
```

复合赋值表达式的值始终是 `()`，而不是所存入的值。

| 类别 | 特型名称 | 表达式 | 等效表达式 |
|------|----------|--------|-----------|
| 算术 | `AddAssign`, `SubAssign`, `MulAssign`, `DivAssign`, `RemAssign` | `x += y` 等 | `x.add_assign(y)` 等 |
| 按位 | `BitAndAssign`, `BitOrAssign`, `BitXorAssign`, `ShlAssign`, `ShrAssign` | `x &= y` 等 | `x.bitand_assign(y)` 等 |

### Complex 的 AddAssign 实现

```rust
use std::ops::AddAssign;

impl<T> AddAssign for Complex<T>
where
    T: AddAssign<T>,
{
    fn add_assign(&mut self, rhs: Complex<T>) {
        self.re += rhs.re;
        self.im += rhs.im;
    }
}
```

> **重要**：复合赋值特型完全独立于对应的二元运算符特型。实现 `Add` 不会自动获得 `AddAssign`，必须自行实现。

## 四、相等性比较（PartialEq 与 Eq）

### 4.1 PartialEq 特型

`x == y` 和 `x != y` 是对 `PartialEq` 的 `eq` 和 `ne` 方法的简写：

```rust
trait PartialEq<Rhs = Self>
where
    Rhs: ?Sized,
{
    fn eq(&self, other: &Rhs) -> bool;
    fn ne(&self, other: &Rhs) -> bool {
        !self.eq(other)
    }
}
```

- 只需定义 `eq`，`ne` 有默认实现（取反 `eq`）
- 通过**引用**获取操作数，不会导致非 `Copy` 类型被移动
- `Rhs: ?Sized` 限界允许与无固定大小类型（如 `str`、`[T]`）进行比较

手动实现示例：

```rust
impl<T: PartialEq> PartialEq for Complex<T> {
    fn eq(&self, other: &Complex<T>) -> bool {
        self.re == other.re && self.im == other.im
    }
}
```

更常见的做法是通过 `derive` 自动派生：

```rust
#[derive(Clone, Copy, Debug, PartialEq)]
struct Complex<T> { ... }
```

### 4.2 部分相等 vs 完全相等

传统数学中等价关系的三个要求：
1. 对称性：`x == y` → `y == x`
2. 传递性：`x == y && y == z` → `x == z`
3. 自反性：`x == x` 始终为真

浮点类型的 `NaN` 不满足第 3 条要求——`NaN != NaN`：

```rust
assert_eq!(0.0 / 0.0 == 0.0 / 0.0, false);
assert!(f64::is_nan(0.0 / 0.0));
```

因此 Rust 使用 `PartialEq` 表示"部分相等"，而 `Eq` 表示"完全相等"：

```rust
trait Eq: PartialEq<Self> {}
```

`Eq` 是 `PartialEq` 的扩展且无新方法——实现它仅表示承诺 `x == x` 始终为真。`f32` 和 `f64` 是标准库中仅有的实现了 `PartialEq` 但未实现 `Eq` 的类型。

手动实现 `Eq`：

```rust
impl<T: Eq> Eq for Complex<T> {}
```

或通过 `derive`：

```rust
#[derive(Clone, Copy, Debug, Eq, PartialEq)]
struct Complex<T> { ... }
```

> 派生 `Eq` 时，如果类型参数没有实现 `Eq`（如 `f32`），则生成的实现也会相应降级。`Complex<f32>` 仅实现 `PartialEq`，`Complex<i32>` 则同时实现 `Eq`。

## 五、有序比较（PartialOrd 与 Ord）

### 5.1 PartialOrd 特型

`<`、`>`、`<=`、`>=` 四个运算符由 `std::cmp::PartialOrd` 统一定义：

```rust
trait PartialOrd<Rhs = Self>: PartialEq<Rhs>
where
    Rhs: ?Sized,
{
    fn partial_cmp(&self, other: &Rhs) -> Option<Ordering>;
    fn lt(&self, other: &Rhs) -> bool { ... }
    fn le(&self, other: &Rhs) -> bool { ... }
    fn gt(&self, other: &Rhs) -> bool { ... }
    fn ge(&self, other: &Rhs) -> bool { ... }
}
```

- `PartialOrd<Rhs>` 扩展了 `PartialEq<Rhs>`：可比较顺序的类型必先可比较相等性
- 只需实现 `partial_cmp`，其他 4 个方法有默认定义
- 返回 `Option<Ordering>`：`None` 表示两者不可比较（如 NaN）

### 5.2 Ordering 枚举

```rust
enum Ordering {
    Less,       // self < other
    Equal,      // self == other
    Greater,    // self > other
}
```

### 5.3 运算符与 PartialOrd 方法对应关系

| 表达式 | 方法调用 | 默认定义 |
|--------|---------|---------|
| `x < y` | `x.lt(y)` | `x.partial_cmp(&y) == Some(Less)` |
| `x > y` | `x.gt(y)` | `x.partial_cmp(&y) == Some(Greater)` |
| `x <= y` | `x.le(y)` | `matches!(x.partial_cmp(&y), Some(Less \| Equal))` |
| `x >= y` | `x.ge(y)` | `matches!(x.partial_cmp(&y), Some(Greater \| Equal))` |

### 5.4 Interval<T> 的部分排序示例

定义一个区间类型，如果一个区间完全落在另一个之前则认为是"小于"关系，重叠时则无序：

```rust
#[derive(Debug, PartialEq)]
struct Interval<T> {
    lower: T, // 闭区间
    upper: T, // 开区间
}

use std::cmp::{Ordering, PartialOrd};

impl<T: PartialOrd> PartialOrd<Interval<T>> for Interval<T> {
    fn partial_cmp(&self, other: &Interval<T>) -> Option<Ordering> {
        if self == other {
            Some(Ordering::Equal)
        } else if self.lower >= other.upper {
            Some(Ordering::Greater)
        } else if self.upper <= other.lower {
            Some(Ordering::Less)
        } else {
            None  // 有重叠，视为不可比较
        }
    }
}

// 使用
assert!(Interval { lower: 10, upper: 20 } <  Interval { lower: 20, upper: 40 });
let left  = Interval { lower: 10, upper: 30 };
let right = Interval { lower: 20, upper: 40 };
assert!(!(left < right));  // 重叠，无顺序关系
```

### 5.5 Ord 特型（完全排序）

```rust
trait Ord: Eq + PartialOrd<Self> {
    fn cmp(&self, other: &Self) -> Ordering;
}
```

- `cmp` 直接返回 `Ordering`（而非 `Option<Ordering>`），总是能确定顺序
- 几乎所有实现了 `PartialOrd` 的类型都应实现 `Ord`；`f32` 和 `f64` 是例外
- 当需要完全排序（如标准库的排序方法）时，`Ord` 是必需的

`Reverse<T>` 包装器：对任意 `Ord` 类型反转排序方向：

```rust
intervals.sort_by_key(|i| Reverse(i.lower));
// 或按上限排序
intervals.sort_by_key(|i| i.upper);
```

## 六、Index 与 IndexMut

通过实现 `Index` 和 `IndexMut`，可使自定义类型支持 `a[i]` 索引表达式。

```rust
trait Index<Idx> {
    type Output: ?Sized;
    fn index(&self, index: Idx) -> &Self::Output;
}

trait IndexMut<Idx>: Index<Idx> {
    fn index_mut(&mut self, index: Idx) -> &mut Self::Output;
}
```

- `a[i]` 等价于 `*a.index(i)`
- `a[i] = z` 或 `&mut a[i]` 等价于 `*a.index_mut(i)`
- 可通过不同 `Idx` 类型支持多种索引方式（切片对 `usize` 和 `Range<usize>` 都实现了 `Index`）
- `HashMap<&str, i32>` 实现了 `Index<&str>`，支持 `m["key"]` 访问

### Image<P> 示例（二维索引）

```rust
struct Image<P> {
    width: usize,
    pixels: Vec<P>,
}

impl<P> std::ops::Index<usize> for Image<P> {
    type Output = [P];
    fn index(&self, row: usize) -> &[P] {
        let start = row * self.width;
        &self.pixels[start..start + self.width]
    }
}

impl<P> std::ops::IndexMut<usize> for Image<P> {
    fn index_mut(&mut self, row: usize) -> &mut [P] {
        let start = row * self.width;
        &mut self.pixels[start..start + self.width]
    }
}

// 使用：image[row][column] = ...;
```

> **限制**：`IndexMut` 只能返回对已有值的可变引用，因此无法用 `m["key"] = val` 向 `HashMap` 插入新条目。这需要先创建默认值的条目再返回引用，但并非所有类型都有低开销的默认值。

## 七、不可重载的运算符

以下运算符**不能**被重载：

| 运算符 | 原因 |
|--------|------|
| `?` | 仅适用于 `Result` 和 `Option` |
| `&&`, `\|\|` | 仅限于 `bool` 值（短路求值语义） |
| `..`, `..=` | 总是创建范围结构体 |
| `&` | 总是借用引用 |
| `=` | 总是移动或复制值 |
| `f(x)` | 函数调用不可重载；需要可调用值时使用闭包（见 `Fn`/`FnMut`/`FnOnce`） |

> **注意**：解引用运算符 `*val` 和点运算符 `.` 可以通过 `Deref`/`DerefMut` 特型重载，将在[内存管理特型](内存管理特型.md)中介绍。
