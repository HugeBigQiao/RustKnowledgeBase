# 泛型

泛型是 Rust 中多态的第二种形式。与 C++ 模板一样，泛型函数/类型可与不同类型一起使用，编译器通过**单态化**为每种具体类型生成独立机器码。

## 一、泛型函数

```rust
fn min<T: Ord>(value1: T, value2: T) -> T {
    if value1 <= value2 { value1 } else { value2 }
}
```

- `<T: Ord>` 是**类型参数**，`Ord` 是**限界**（bound），限制 T 必须是可比较的有序类型
- 编译器为每种实际类型单独生成代码（单态化）

### 类型推断

```rust
min(10, 20);         // Rust 推断 T = i32
min("abc", "xyz");   // Rust 推断 T = &str
```

需要时显式指定：

```rust
min::<i32>(10, 20);
let v: Vec<i32> = (0..1000).collect();          // 错误：无法推断
let v = (0..1000).collect::<Vec<i32>>();         // 正确
```

## 二、多个限界

```rust
use std::hash::Hash;
use std::fmt::Debug;

fn top_ten<T: Debug + Hash + Eq>(values: &Vec<T>) { /* ... */ }
```

`+` 表示同时需要多个特型。

## 三、where 子句

当限界较多时，使用 `where` 提升可读性：

```rust
fn run_query<M, R>(data: &DataSet, map: M, reduce: R) -> Results
where
    M: Mapper + Serialize,
    R: Reducer + Serialize,
{ /* ... */ }
```

`where` 也可用于结构体、枚举、类型别名和方法的限界。

## 四、泛型结构体

```rust
pub struct Queue<T> {
    older: Vec<T>,
    younger: Vec<T>,
}

impl<T> Queue<T> {
    pub fn new() -> Self {
        Queue { older: Vec::new(), younger: Vec::new() }
    }
    pub fn push(&mut self, t: T) {
        self.younger.push(t);
    }
}

// 特定类型的专属实现
impl Queue<f64> {
    fn sum(&self) -> f64 { /* ... */ }
}
```

泛型枚举同理：`Option<T>`、`Result<T, E>`、`BinaryTree<T>`。

## 五、泛型方法（非泛型类型上）

```rust
impl PancakeStack {
    fn push<T: Topping>(&mut self, goop: T) -> PancakeResult<()> {
        goop.pour(&self);
        self.absorb_topping(goop)
    }
}
```

## 六、泛型类型别名

```rust
type PancakeResult<T> = Result<T, PancakeError>;
```

## 七、生命周期与常量参数

```rust
// 生命周期参数在前，类型参数在后
fn nearest<'t, 'c, P>(target: &'t P, candidates: &'c [P]) -> &'c P
where P: MeasureDistance
{ /* ... */ }

// 常量泛型参数
fn dot_product<const N: usize>(a: [f64; N], b: [f64; N]) -> f64 {
    let mut sum = 0.;
    for i in 0..N { sum += a[i] * b[i]; }
    sum
}

dot_product::<3>([0.2, 0.4, 0.6], [0., 0., 1.]);
dot_product([3., 4.], [-5., 1.]);  // 自动推断 N = 2
```

生命周期不会影响机器码，只有不同**类型**才会生成多个副本。

## 八、impl Trait（返回位置的 impl Trait）

简化复杂返回类型（如迭代器组合器），保持静态派发：

```rust
// 丑陋的完整类型签名
fn cyclical_zip(v: Vec<u8>, u: Vec<u8>) ->
    iter::Cycle<iter::Chain<IntoIter<u8>, IntoIter<u8>>>
{
    v.into_iter().chain(u.into_iter()).cycle()
}

// 简洁的 impl Trait
fn cyclical_zip(v: Vec<u8>, u: Vec<u8>) -> impl Iterator<Item = u8> {
    v.into_iter().chain(u.into_iter()).cycle()
}
```

- 返回类型只说明意图，隐藏实现细节
- 静态派发，无动态派发开销，无堆分配
- 库作者可自由更换内部实现而不破坏 API

### impl Trait 的限制

不能在不同分支返回不同类型：

```rust
fn make_shape(shape: &str) -> impl Shape {
    match shape {
        "circle" => Circle::new(),
        "triangle" => Triangle::new(),  // 错误：不兼容的类型
        _ => Rectangle::new(),
    }
}
```

若需此行为，必须使用特型对象 `Box<dyn Shape>`。

### 参数位置的 impl Trait

```rust
// 等价写法
fn say_hello(out: &mut impl Write) { /* ... */ }
fn say_hello<W: Write>(out: &mut W) { /* ... */ }
```

参数位置的 `impl Trait` 就是泛型的语法糖。

## 九、单态化与代码膨胀

每次使用不同类型调用泛型函数，编译器都生成一份新的机器码。可能增大二进制文件（**代码膨胀**）。

- 大多数场景影响甚微
- 如需控制，可手动抽取非泛型部分或使用特型对象
- Rust 的优化器可内联并进一步消除不必要的代码
