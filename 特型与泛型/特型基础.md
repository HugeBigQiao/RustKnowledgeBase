# 特型基础

特型（Trait）是 Rust 中的接口或抽象基类，定义类型"能做什么"。特型类似于 Java/C# 中的接口，但受 Haskell 类型类的启发，能力更丰富。

## 一、特型定义

```rust
trait Visible {
    fn draw(&self, canvas: &mut Canvas);
    fn hit_test(&self, x: i32, y: i32) -> bool;
}
```

### 实现特型

```rust
impl Visible for Broom {
    fn draw(&self, canvas: &mut Canvas) {
        for y in self.broomstick_range() {
            canvas.write_at(self.x, y, '|');
        }
        canvas.write_at(self.x, self.y, 'M');
    }

    fn hit_test(&self, x: i32, y: i32) -> bool {
        self.x == x
        && self.y - self.height - 1 <= y
        && y <= self.y
    }
}
```

> **孤儿规则**：只能在类型或特型至少有一个在当前 crate 中定义时才能实现特型。这确保了特型实现的唯一性。

### 特型方法作用域

**特型本身必须在作用域内**，否则其方法不可见：

```rust
let mut buf: Vec<u8> = vec![];
buf.write_all(b"hello")?;   // 错误：需要 use std::io::Write;

use std::io::Write;
buf.write_all(b"hello")?;   // 正确
```

`Clone`、`Iterator` 等通过**预导入（prelude）**自动在作用域中，无需手动导入。

## 二、默认方法

特型可以为方法提供默认实现，实现者可以选择覆盖或保留：

```rust
trait Write {
    fn write(&mut self, buf: &[u8]) -> Result<usize>;
    fn flush(&mut self) -> Result<()>;

    // 默认实现——实现者可覆盖也可不覆盖
    fn write_all(&mut self, buf: &[u8]) -> Result<()> {
        let mut bytes_written = 0;
        while bytes_written < buf.len() {
            bytes_written += self.write(&buf[bytes_written..])?;
        }
        Ok(())
    }
}
```

`Iterator` 特型只有一个必需方法 `.next()`，却有几十个带默认实现的方法。

## 三、特型对象（动态派发）

特型对象通过 `dyn` 关键字使用，在**运行期**派发：

```rust
fn say_hello(out: &mut dyn Write) -> std::io::Result<()> {
    out.write_all(b"hello world\n")?;
    out.flush()
}

// 可接受任意 Write 实现者
say_hello(&mut File::create("hello.txt")?)?;
say_hello(&mut vec![])?;
```

### 特型对象的内存布局

特型对象是**胖指针**（两个机器字）：指向数据的指针 + 指向虚表（vtable）的指针。

```
&mut dyn Write
┌──────────────────┐
│ 数据指针        │ → 具体值（如 Vec<u8>）
│ 虚表指针        │ → Write 的方法表
└──────────────────┘
```

- Rust 的虚表只在编译期生成一次，被同类型的所有对象共享
- 不像 C++，虚表不与结构体本身存储在一起——每个结构体可实现几十个特型而无 vptr 开销
- 自动将 `&mut File` 转换为 `&mut dyn Write`（前提是 `File: Write`）

### `dyn` 与指针类型

```rust
let w: Box<dyn Write> = Box::new(local_file);  // 拥有型特型对象
let r: Rc<dyn Write> = Rc::new(some_writer);    // 引用计数特型对象
```

### 特型对象的限制

- `dyn Write` **不能作为变量类型**（大小不固定），必须通过引用/指针
- 使用 `Self` 的特型不兼容特型对象（Rust 无法在编译期验证类型一致性）
- 类型关联函数（无 `self` 的方法）需要 `where Self: Sized` 限界才能兼容

## 四、泛型函数（静态派发）

```rust
fn say_hello<W: Write>(out: &mut W) -> std::io::Result<()> {
    out.write_all(b"hello world\n")?;
    out.flush()
}

say_hello(&mut local_file)?;  // 编译器生成 say_hello::<File>
say_hello(&mut bytes)?;       // 编译器生成 say_hello::<Vec<u8>>
```

这称为**单态化**：编译器为每种用到的类型生成独立的机器码。无虚表、无动态派发，可被内联优化至极致。

### 特型对象 vs 泛型选择

| 场景 | 推荐 |
|------|------|
| 混合类型集合 | 特型对象 `Vec<Box<dyn Trait>>` |
| 减少二进制大小 | 特型对象（避免代码膨胀） |
| 追求运行速度 | 泛型（无虚函数开销、可内联） |
| 多个特型限界 | 泛型（`T: A + B + C`） |
| 使用关联函数/Self | 泛型（特型对象不支持） |

Rust 中泛型是更常见的选择。

## 五、扩展特型

为任意类型（包括标准库类型）添加方法：

```rust
trait IsEmoji {
    fn is_emoji(&self) -> bool;
}

impl IsEmoji for char {
    fn is_emoji(&self) -> bool { /* ... */ }
}

assert_eq!('$'.is_emoji(), false);
```

泛型实现——一次性为整个类型家族添加扩展：

```rust
impl<W: Write> WriteHtml for W {
    fn write_html(&mut self, html: &HtmlDocument) -> io::Result<()> {
        // ...
    }
}
```

## 六、特型中的 Self

```rust
pub trait Clone {
    fn clone(&self) -> Self;
}

pub trait Spliceable {
    fn splice(&self, other: &Self) -> Self;  // 只能同类型拼接
}
```

使用了 `Self` 的特型**与特型对象不兼容**——编译器无法在编译期验证 `left` 和 `right` 是否同类型。

## 七、子特型

```rust
trait Creature: Visible {  // Creature 是 Visible 的子特型
    fn position(&self) -> (i32, i32);
    fn facing(&self) -> Direction;
}

// 等价写法
trait Creature where Self: Visible { ... }
```

实现 `Creature` 的类型必须同时实现 `Visible`。子特型不继承超特型的关联项，需要各自在作用域内。

## 八、类型关联函数

特型可以包含不取 `self` 的关联函数（相当于静态方法/构造函数）：

```rust
trait StringSet {
    fn new() -> Self;
    fn from_slice(strings: &[&str]) -> Self;
    fn contains(&self, string: &str) -> bool;
    fn add(&mut self, string: &str);
}

fn unknown_words<S: StringSet>(document: &[String], wordlist: &S) -> S {
    let mut unknowns = S::new();
    for word in document {
        if !wordlist.contains(word) { unknowns.add(word); }
    }
    unknowns
}
```

特型对象不支持关联函数，需加 `where Self: Sized` 限界豁免。

## 九、完全限定的方法调用

```rust
"hello".to_string()              // 常规方法调用
str::to_string("hello")          // 作为关联函数
ToString::to_string("hello")     // 指定特型
<str as ToString>::to_string("hello")  // 完全限定
```

使用场景：
- **方法名冲突**：`Visible::draw(&outlaw)` vs `HasPistol::draw(&outlaw)`
- **无法推断类型**：`i64::abs(zero)`
- **将方法作为函数值传递**：`.map(ToString::to_string)`
- **宏中调用特型方法**
