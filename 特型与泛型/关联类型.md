# 关联类型与泛型特型

关联类型（Associated Type）和泛型特型是 Rust 描述类型之间关系的两种方式。关联类型适合"每种实现有**一个**特定关联类型"的场景，泛型特型适合"每种实现可与**多种**其他类型配合"的场景。

## 一、关联类型

### 定义

```rust
pub trait Iterator {
    type Item;                           // 关联类型

    fn next(&mut self) -> Option<Self::Item>;
    // ...
}
```

每个 `Iterator` 实现必须指定 `Item` 是什么类型：

```rust
impl Iterator for Args {
    type Item = String;                  // 设定关联类型

    fn next(&mut self) -> Option<String> {
        // ...
    }
}
```

### 泛型代码中使用

```rust
fn collect_into_vector<I: Iterator>(iter: I) -> Vec<I::Item> {
    let mut results = Vec::new();
    for value in iter {
        results.push(value);
    }
    results
}
```

### 为关联类型加限界

```rust
use std::fmt::Debug;

fn dump<I>(iter: I)
where
    I: Iterator,
    I::Item: Debug,            // 限制关联类型
{
    for (index, value) in iter.enumerate() {
        println!("{}: {:?}", index, value);
    }
}

// 等价写法：直接指定关联类型
fn dump(iter: &mut dyn Iterator<Item = String>) {
    for (index, s) in iter.enumerate() {
        println!("{}: {:?}", index, s);
    }
}
```

### 更多示例

```rust
trait Pattern {
    type Match;
    fn search(&self, string: &str) -> Option<Self::Match>;
}

impl Pattern for char {
    type Match = usize;   // 字符模式：匹配位置

    fn search(&self, string: &str) -> Option<usize> {
        string.find(*self)
    }
}

// 正则表达式模式的 Match 可能是包含开始/结束/分组信息的结构体
```

### 关联类型 vs 泛型参数

| 特性 | 关联类型 | 泛型类型参数 |
|------|---------|-------------|
| 每实现数量 | 最多一个 | 多个 |
| 典型场景 | `Iterator` 的 `Item` | `Mul<RHS>` 的右操作数 |
| 使用方式 | `I::Item` | `Mul<f64>`, `Mul<i32>` |
| 调用方指定 | 否（实现决定） | 是（调用方选择） |

**何时用关联类型**：每种实现只有一个自然的关联类型（每种迭代器生成一种类型，每种模式有一种匹配类型）。

**何时用泛型特型**：实现需要与多种不同类型组合（如一个类型既能与 `f64` 乘，也能与 `i32` 乘）。

## 二、泛型特型

### 定义

```rust
pub trait Mul<RHS = Self> {     // RHS 默认 = Self
    type Output;

    fn mul(self, rhs: RHS) -> Self::Output;
}
```

- `Mul<RHS>` 是一个泛型特型，`Mul<f64>`、`Mul<String>` 是不同的特型
- `RHS = Self` 是默认类型参数，`impl Mul for Complex` 等价于 `impl Mul<Complex> for Complex`

### 多实现

同一个类型可实现多次泛型特型（不同类型参数）：

```rust
impl Mul<f64> for WindowSize {
    type Output = WindowSize;
    fn mul(self, rhs: f64) -> WindowSize { /* ... */ }
}

impl Mul<i32> for WindowSize {
    type Output = WindowSize;
    fn mul(self, rhs: i32) -> WindowSize { /* ... */ }
}
```

### 孤儿规则的特殊豁免

可为外部类型实现外部特型，只要特型的**类型参数之一**在当前 crate 中定义：

```rust
// 即使 Mul 和 f64 都来自标准库，WindowSize 是你定义的
impl Mul<WindowSize> for f64 { /* ... */ }
impl<T> Mul<WindowSize> for Vec<T> { /* ... */ }
```

这是因为其他 crate 不可能定义 `Mul<WindowSize>`，不会有冲突。

## 三、总结：模式与迭代器

关联类型和泛型特型构成了 Rust 特型系统的核心，理解它们的区别至关重要：

- **`Iterator<Item = T>`**：迭代器的元素类型由实现决定——关联类型
- **`Mul<RHS>`**：乘法右操作数的类型由调用方决定——泛型类型参数

两者可组合使用（如 `Mul` 同时有泛型参数 `RHS` 和关联类型 `Output`）。
