# 内存管理特型

标准库中与**内存生命周期、所有权、智能指针**相关的特型：`Drop`（析构）、`Sized`（固定大小）、`Clone`（显式克隆）、`Copy`（按位复制）、`Deref`/`DerefMut`（解引用）。

> 类型转换相关的特型（`Default`、`From`/`Into`、`AsRef`/`AsMut`、`Borrow`/`BorrowMut`、`Cow` 等）请参见：[类型转换特型](类型转换特型.md)。

## 特型速查表

| 特型 | 描述 |
|------|------|
| `Drop` | 析构器。值被丢弃时自动运行的清理代码 |
| `Sized` | 编译期已知固定大小的标记特型（自动实现，不能手动实现） |
| `Clone` | 支持显式克隆值的类型 |
| `Copy` | 可通过逐字节复制来克隆的标记特型 |
| `Deref`/`DerefMut` | 智能指针类型，重载 `*` 和 `.` 运算符 |

---

## 一、Drop——析构器

```rust
trait Drop {
    fn drop(&mut self);
}
```

当一个值的拥有者消失时，Rust 会**丢弃**该值。如果该类型实现了 `Drop`，Rust 会先调用 `drop` 方法，然后再丢弃字段或元素。

### 使用场景

- 绝大多数情况下**不需要**手动实现 `Drop`——Rust 会自动处理
- 当类型拥有 Rust 不了解的资源时才需要（如操作系统文件描述符、C 库句柄）：

```rust
impl Drop for FileDesc {
    fn drop(&mut self) {
        let _ = unsafe { libc::close(self.fd) };
    }
}
```

### 关键规则

- **不能显式调用** `drop` 方法——Rust 会标记为错误
- 如果类型实现了 `Drop`，就**不能再实现 `Copy`**（多次 drop 同一份数据是错的）
- 值移动走后，Rust 使用不可见标志跟踪状态，确保值仅被丢弃一次
- 标准库 `drop` 函数：`fn drop<T>(_x: T) {}`——获取所有权然后什么都不做

```rust
// 被移动的值不会被drop两次
let p;
{
    let q = Appellation { name: "...".to_string(), nicknames: vec![...] };
    if complicated_condition() {
        p = q;  // q的值移动到p
    }
}  // 如果q已被移走，则不会drop它
// p在此处离开作用域时被drop
```

## 二、Sized——固定大小类型

**固定大小类型**：每个值在内存中有相同大小的类型（如 `u64` 占 8 字节，`Vec<T>` 占三个指针宽度）。

**无固定大小类型**：值的大小不尽相同，如：
- `str`（字符串切片）、`[T]`（数组切片）
- `dyn Trait`（特型对象）

### 隐式 Sized 限界

```rust
struct S<T> { ... }         // 实际是 struct S<T: Sized> { ... }
struct S<T: ?Sized> { ... } // 排除Sized要求（"大小不确定"）
```

- 无固定大小的值不能存储在变量中或按值传递，只能通过**胖指针**（两个机器字宽）操作
- 指向切片的指针携带长度，特型对象携带虚表指针
- 结构体的最后一个字段可以是无固定大小的，这让结构体本身也变成无固定大小的

```rust
// RcBox的简化定义——最后一个字段value可以是无固定大小的
struct RcBox<T: ?Sized> {
    ref_count: usize,
    value: T,
}
```

## 三、Clone——显式克隆

```rust
trait Clone: Sized {
    fn clone(&self) -> Self;
    fn clone_from(&mut self, source: &Self) {
        *self = source.clone()
    }
}
```

- `clone` 方法构造 `self` 的独立副本并返回
- 克隆通常是昂贵的（需对拥有的值分配副本），因此**不会自动发生**，必须显式调用
- `Rc<T>` 和 `Arc<T>` 例外：克隆只增加引用计数
- `clone_from` 可优化重用已有资源——泛型代码中应优先使用
- 通过 `#[derive(Clone)]` 自动派生

```rust
// clone_from的优化：如果已有缓冲区容量足够，无需重新分配
// 泛型代码中优先使用：
target.clone_from(&source);
// 而不是：
target = source.clone();
```

无法合理复制的类型不实现 `Clone`（如 `Mutex`、`File`）。`File` 提供了可失败的 `try_clone`。

## 四、Copy——按位复制标记

```rust
trait Copy: Clone { }
```

实现 `Copy` 意味着赋值时执行浅层的逐字节复制，而非移动。限制：
- 只能用于**不拥有堆资源、不拥有OS句柄**的类型
- 实现了 `Drop` 的类型**不能**是 `Copy`
- 通过 `#[derive(Copy, Clone)]` 自动派生

> 让类型成为 `Copy` 需慎重：使类型更易用，但施加严格限制。隐式复制大类型也可能影响性能。

## 五、Deref 与 DerefMut——解引用

```rust
trait Deref {
    type Target: ?Sized;
    fn deref(&self) -> &Self::Target;
}

trait DerefMut: Deref {
    fn deref_mut(&mut self) -> &mut Self::Target;
}
```

### 功能一：重载 `*` 运算符

```rust
// Selector<T> 表现得像指向当前元素的指针
impl<T> Deref for Selector<T> {
    type Target = T;
    fn deref(&self) -> &T {
        &self.elements[self.current]
    }
}

impl<T> DerefMut for Selector<T> {
    fn deref_mut(&mut self) -> &mut T {
        &mut self.elements[self.current]
    }
}

// 使用
let mut s = Selector { elements: vec!['x', 'y', 'z'], current: 2 };
assert_eq!(*s, 'z');
*s = 'w';
```

### 功能二：隐式解引用（Deref 强制转换）

如果插入一个 `deref` 调用就能解决类型不匹配，Rust 会**自动插入**它：

- `Rc<String>` 上调用 `String` 方法：`r.find('?')`（自动解引用成 `&String`）
- `Vec<u8>` 传入需要 `&[u8]` 的函数：直接用 `&v`
- Rust 会**连续应用多个**隐式解引用（如 `&Rc<String>` → `&String` → `&str`）

```rust
fn show_it(thing: &str) { println!("{}", thing); }
let s = Selector { elements: vec!["good", "bad", "ugly"], current: 2 };
show_it(&s);  // OK：隐式解引用：&Selector<&str> → &str
```

### 重要限制

**隐式解引用不会用于满足类型变量的限界**：

```rust
fn show_it_generic<T: Display>(thing: T) { ... }
show_it_generic(&s);  // 错误！不会通过解引用来满足 T: Display 限界

// 解决方案：
show_it_generic(&s as &str);  // 或者：
show_it_generic(&*s);
```

### 设计原则

`Deref`/`DerefMut` 应仅用于**智能指针类型**（`Box`、`Rc`、`Arc`）及其拥有型版本的惯用引用（`Vec<T>` → `[T]`，`String` → `str`）。不应仅为让目标类型的方法可见而实现它们（类似 C++ 的继承）。
