# 类型转换特型

标准库中与**默认值、引用转换、值转换、写时克隆**相关的特型：`Default`、`AsRef`/`AsMut`、`Borrow`/`BorrowMut`、`From`/`Into`、`TryFrom`/`TryInto`、`ToOwned`、`Cow`。

> 内存生命周期相关的特型（`Drop`、`Sized`、`Clone`、`Copy`、`Deref`/`DerefMut`）请参见：[内存管理特型](内存管理特型.md)。

## 特型速查表

| 特型 | 描述 |
|------|------|
| `Default` | 具有合理"默认值"的类型 |
| `AsRef`/`AsMut` | 从一种类型借入另一种类型的引用 |
| `Borrow`/`BorrowMut` | 与 `AsRef` 类似，但额外保证哈希、排序和相等性一致 |
| `From`/`Into` | 值转换（获取所有权） |
| `TryFrom`/`TryInto` | 可能失败的值转换 |
| `ToOwned` | 将引用转换为拥有型值 |
| `Cow` | 写时克隆——延迟分配的智能借用 |

## 一、Default——默认值

```rust
trait Default {
    fn default() -> Self;
}
```

所有 Rust 集合类型实现了 `Default`，返回空集合。常见用途：

### 与 `partition` 等迭代器方法配合

```rust
let (upper, lower): (String, String)
    = "Great Teacher Onizuka".chars().partition(|&c| c.is_uppercase());
assert_eq!(upper, "GTO");
assert_eq!(lower, "reat eacher nizuka");
```

### 结构体更新语法

```rust
let params = glium::DrawParameters {
    line_width: Some(0.02),
    point_size: Some(0.02),
    .. Default::default()  // 其余字段取默认值
};
```

- 通过 `#[derive(Default)]` 自动派生（需所有字段实现 `Default`）
- 智能指针 `Rc<T>`、`Box<T>` 等会自动获得 `Default` 实现（当 `T: Default` 时）

## 二、AsRef 与 AsMut——引用转换

```rust
trait AsRef<T: ?Sized> {
    fn as_ref(&self) -> &T;
}

trait AsMut<T: ?Sized> {
    fn as_mut(&mut self) -> &mut T;
}
```

### 让函数参数更灵活

```rust
fn open<P: AsRef<Path>>(path: P) -> Result<File>

// 可接受 String, &str, OsString, OsStr, PathBuf, Path 等
let f = std::fs::File::open("/home/jimb/.emacs")?;
```

### 通用实现链

标准库有通用实现：`T: AsRef<U>` → `&T: AsRef<U>`。所以 `str: AsRef<Path>` 自动意味着 `&str: AsRef<Path>`。

### AsMut 的不变性约束

只有修改 `&mut T` 不违反类型不变性时，才应实现 `AsMut<T>`。例如 `String` 实现了 `AsRef<[u8]>` 但**不**实现 `AsMut<[u8]>`——因为那可能破坏 UTF-8 编码保证。

> 优先使用 `AsRef<Foo>` 而非自定义的 `AsFoo` 特型。

## 三、Borrow 与 BorrowMut——哈希一致的借用

```rust
trait Borrow<Borrowed: ?Sized> {
    fn borrow(&self) -> &Borrowed;
}

trait BorrowMut<Borrowed: ?Sized>: Borrow<Borrowed> {
    fn borrow_mut(&mut self) -> &mut Borrowed;
}
```

`Borrow` 比 `AsRef` 更严格：借出的 `&T` 必须与原始值以**相同方式**进行哈希和比较。

### 为什么需要 Borrow？

`String` 实现了三个不同的 `AsRef`：`AsRef<str>`、`AsRef<[u8]>`、`AsRef<Path>`。但这三者的哈希值不同，只有 `&str` 的哈希与 `String` 一致。因此 `String` 只实现了 `Borrow<str>`。

### HashMap::get 的签名

```rust
impl<K, V> HashMap<K, V> where K: Eq + Hash {
    fn get<Q: ?Sized>(&self, key: &Q) -> Option<&V>
        where K: Borrow<Q>,
              Q: Eq + Hash
    { ... }
}
```

这允许用 `&str` 在 `HashMap<String, i32>` 中查找，而无需分配 `String`：

```rust
let m: HashMap<String, i32> = ...;
m.get("twenty-two");  // 直接传 &str，无需 to_string()
```

- 每种类型都能从自身借用：`T: Borrow<T>`
- `&mut T` 也实现了 `Borrow<T>`（返回共享引用）

## 四、From 与 Into——值转换

```rust
trait Into<T>: Sized {
    fn into(self) -> T;
}

trait From<T>: Sized {
    fn from(other: T) -> Self;
}
```

### Into：让函数参数灵活

```rust
fn ping<A>(address: A) -> std::io::Result<bool>
    where A: Into<Ipv4Addr>
{
    let ipv4_address = address.into();
    ...
}

ping(Ipv4Addr::new(23, 21, 68, 141));  // Ipv4Addr
ping([66, 146, 219, 98]);               // [u8; 4]
ping(0xd076eb94_u32);                    // u32
```

### From：泛型构造函数

```rust
let addr1 = Ipv4Addr::from([66, 146, 219, 98]);
let addr2 = Ipv4Addr::from(0xd076eb94_u32);
```

- 每当你定义单参数构造函数时，应实现为 `From<T>`
- 实现 `From` 自动获得对应的 `Into`
- 每种类型 `T` 自动实现 `From<T>` 和 `Into<T>`

### 零成本转换

```rust
let text = "Beautiful Soup".to_string();
let bytes: Vec<u8> = text.into();
```

`String` → `Vec<u8>` 的实现直接复用堆缓冲区，无需分配或复制。

### 与 `?` 运算符的配合

标准库提供了通用实现，将任何 `Error` 类型转换为 `Box<dyn Error>`：

```rust
impl<'a, E: Error + Send + Sync + 'a> From<E>
    for Box<dyn Error + Send + Sync + 'a> { ... }

// 于是 ? 运算符自动处理错误类型转换：
fn parse_i32_bytes(b: &[u8]) -> GenericResult<i32> {
    Ok(std::str::from_utf8(b)?.parse::<i32>()?)
}
```

## 五、TryFrom 与 TryInto——可失败转换

```rust
pub trait TryFrom<T>: Sized {
    type Error;
    fn try_from(value: T) -> Result<Self, Self::Error>;
}

pub trait TryInto<T>: Sized {
    type Error;
    fn try_into(self) -> Result<T, Self::Error>;
}
```

用于可能失败的转换。Rust 没有为会丢失信息的数值转换实现 `From`（如 `i64` → `i32`），而是使用 `TryFrom`：

```rust
let huge = 2_000_000_000_000i64;
// i32 实现了 TryFrom<i64>
let smaller: i32 = huge.try_into().unwrap_or(i32::MAX);
```

### 自定义 TryFrom 示例

```rust
impl TryInto<LinearShift> for Transform {
    type Error = TransformError;

    fn try_into(self) -> Result<LinearShift, Self::Error> {
        if !self.normalized() {
            return Err(TransformError::NotNormalized);
        }
        ...
    }
}
```

## 六、ToOwned——引用转拥有型值

```rust
trait ToOwned {
    type Owned: Borrow<Self>;
    fn to_owned(&self) -> Self::Owned;
}
```

与 `Clone` 不同，`to_owned` 可返回不同于 `Self` 的类型：

- `str` 实现 `ToOwned<Owned=String>`——`"hello".to_owned()` 返回 `String`
- `[T]` 实现 `ToOwned<Owned=Vec<T>>`（当 `T: Clone` 时）
- `Path` 实现 `ToOwned<Owned=PathBuf>`

## 七、Cow——写时克隆

```rust
enum Cow<'a, B: ?Sized>
    where B: ToOwned
{
    Borrowed(&'a B),
    Owned(<B as ToOwned>::Owned),
}
```

`Cow<B>` 要么借入共享引用，要么拥有可借入该引用的值。实现 `Deref`，可像 `&B` 一样使用。

核心方法：
- `to_mut()` → `&mut B`：如果是 `Borrowed`，先调用 `to_owned` 克隆，再返回可变引用（"写入时克隆"）
- `into_owned()` → 拥有型值：必要时克隆，转移所有权

### 典型用途：返回静态字符串或计算字符串

```rust
use std::borrow::Cow;

fn describe(error: &Error) -> Cow<'static, str> {
    match *error {
        Error::OutOfMemory => "out of memory".into(),  // Cow::Borrowed
        Error::FileNotFound(ref path) => {
            format!("file not found: {}", path.display()).into()  // Cow::Owned
        }
    }
}

// 调用者无需分配即可使用：
println!("{}", describe(&error));  // 当作 &str 使用

// 需要拥有型值时再分配：
log.push(describe(&error).into_owned());
```

使用 `Cow` 可将分配推迟到确有必要时。
