# Rust 引用与借用

引用允许在不转移所有权的情况下访问数据。创建引用的行为叫**借用**。引用是一个指向变量地址的指针，但比原始指针安全，因为 Rust 在编译时强制保证引用的有效性。

## 一、引用规则速览

- 在任意给定时间，要么只能有一个可变引用，要么可以有任意数量的不可变引用
- 引用必须始终有效（不能悬垂）

## 二、不可变引用 `&T`

不可变引用允许读取数据，但不能修改。

```rust
    fn print_length(s: &String) {
        println!("{}", s);
    }

    let s = String::from("hello");
    print_length(&s);  // 传入不可变引用，不转移所有权
    // s 依然可用

    let len = s.len();  // .len() 内部也接受 &self
```

在一个作用域内可以同时存在多个不可变引用，因为它们都不会修改数据。

```rust
    let s = String::from("hello");
    let r1 = &s;
    let r2 = &s;
    println!("{} {}", r1, r2);  // 两个不可变引用同时存在
```

## 三、可变引用 `&mut T`

可变引用允许修改被借用的数据，但有严格限制：同一时刻只能有一个可变引用，不能与任何不可变引用共存。

```rust
    let mut s = String::from("hello");
    let r = &mut s;
    r.push_str(" world");  // 通过可变引用修改 String
    println!("{}", r);

    // 在同一作用域中不能混用可变与不可变引用：
    let mut s = String::from("hello");
    let r1 = &s;
    // let r2 = &mut s;  // 错误：已有不可变引用时不能创建可变引用
    println!("{}", r1);   // r1 最后一次使用后，它的借用结束

    // 以下可行：
    let mut s = String::from("hello");
    let r1 = &s;
    println!("{}", r1);  // r1 用完
    let r2 = &mut s;     // 此时 r1 已失效，可变借用可以创建
    r2.push_str(" world");
```

## 四、引用的作用域与作用域结束

引用的有效期从创建开始，到最后一次使用结束。在借用结束后，才能创建新的不同类引用。

```rust
    let mut s = String::from("hello");
    {
        let r1 = &s;      // 不可变借用开始
        println!("{}", r1); // r1 最后使用，借用结束
    }
    // r1 已离开作用域，此处可创建可变借用
    let r2 = &mut s;
    r2.push_str(" world");
```

## 五、悬垂引用：编译器保证安全

Rust 编译器会阻止返回局部变量的引用，因为那样会产生悬垂指针。

```rust
    // 错误示例：返回局部变量的引用
    // fn dangle() -> &String {
    //     let s = String::from("hello");
    //     &s  // s 在此被释放，返回引用无效
    // }

    // 正确方式：返回字符串本身（所有权转移）
    fn no_dangle() -> String {
        let s = String::from("hello");
        s
    }
```

## 六、借用与切片的关系

切片 `&[T]` 和 `&str` 本身就是一种借用，它们是对已有连续数据的部分引用。切片借用遵循同样的不可变/可变规则。

```rust
    let mut arr = [1, 2, 3, 4, 5];
    let slice = &arr[1..3];  // 不可变借用
    // arr[1] = 10;  // 错误：已有不可变借用
    println!("{:?}", slice);
    // slice 借用结束，此后可修改
    arr[1] = 10;
    println!("{:?}", arr);
```
