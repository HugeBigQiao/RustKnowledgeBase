# Rust 所有权

所有权是 Rust 最独特的特性，它在编译时管理内存，无需垃圾回收器或手动分配/释放。理解所有权的三个核心规则，是写出安全、高效 Rust 代码的基础。

## 一、所有权规则速览

- Rust 中每一个值都有一个被称为**所有者**的变量
- 值在任一时刻**只能有一个所有者**
- 当所有者离开作用域，值将被**自动丢弃**（调用 `drop`）

## 二、栈与堆：所有权管理的内存基础

所有权系统的设计直接针对堆内存的管理。数据存储在栈还是堆上，决定了它在所有权转移时的行为。

- **栈**：存放大小编译时已知的固定大小数据（如整数、浮点、布尔、定长元组）。栈上数据的拷贝非常廉价，直接按位复制。
- **堆**：存放大小编译时未知或可能变化的数据（如 `String`、`Vec<T>`、`Box<T>`）。堆上数据通过指针访问，拷贝代价高，所有权转移涉及指针的移动而非数据复制。

```rust
    // 栈上数据：简单拷贝
    let x = 5;
    let y = x;  // 值被完整复制到 y，x 仍然可用
    println!("x = {}, y = {}", x, y);  // 合法

    // 堆上数据：所有权移动
    let s1 = String::from("hello");
    let s2 = s1;  // s1 的堆指针被移动到 s2，s1 不再有效
    // println!("{}", s1);  // 编译错误：s1 已被移动
```

## 三、移动 (Move)：所有权的转移

当变量赋值、传递给函数、或从函数返回时，所有权可能会转移。

```rust
    // 赋值时的移动
    let s1 = String::from("hello");
    let s2 = s1;          // s1 移动到 s2
    // s1 已失效

    // 函数传参时的移动
    fn take_ownership(s: String) {
        println!("{}", s);
    } // s 离开作用域，内存释放

    let s3 = String::from("world");
    take_ownership(s3);   // s3 移动到函数参数
    // s3 已失效
```

## 四、克隆 (Clone)：深拷贝堆数据

如果需要保留原值并产生一份独立的堆数据，需要使用 `clone()` 方法进行深度复制。

```rust
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 堆数据被复制，s1 和 s2 都可用
    println!("s1 = {}, s2 = {}", s1, s2);  // 合法
```

## 五、Copy trait：栈上数据的自动拷贝

拥有 `Copy` trait 的类型在赋值时自动按位复制，不会发生所有权转移。通常存储在栈上的基本类型都实现了 `Copy`，如整数、浮点、布尔、字符、元组（若其所有元素均实现 `Copy`）等。

```rust
    let x = 42;
    let y = x;  // 自动复制，非移动
    println!("{} {}", x, y);  // 两者均可用
```

## 六、移动与控制流

在复杂控制流中使用移动时，一般原则是：如果一个变量的值有可能已被移走，并且尚未明确赋予新值，则被视为未初始化。

### if 表达式中的移动

```rust
let x = vec![10, 20, 30];
if c {
    f(x);   // 可以在此分支中移动 x
} else {
    g(x);   // 也可以在另一个分支中移动 x
}
h(x);   // 错误：任何路径用过 x 后，x 在此处都是未初始化
```

### 循环中的移动

```rust
let x = vec![10, 20, 30];
while f() {
    g(x);   // 错误：x 在第一次迭代中已被移走，第二次迭代是未初始化
}
```

除非在每次迭代中重新赋值：

```rust
let mut x = vec![10, 20, 30];
while f() {
    g(x);       // 移走 x
    x = h();    // 赋予新值，下一次迭代可用
}
e(x);
```

## 七、移动与索引内容

移动会让源变为未初始化，但并非每种拥有者都能标记为未初始化。例如不能直接从 `Vec` 的索引中移出元素：

```rust
let mut v = Vec::new();
for i in 101..106 {
    v.push(i.to_string());
}

let third = v[2];   // 错误：不能移动到 Vec 索引结构之外
let fifth = v[4];   // 同样错误
```

若 Rust 允许这种操作，就需要让向量记录每个元素的初始化状态，代价过高。从向量中移出元素的三种方法：

```rust
// 方法一：从末尾弹出
let fifth = v.pop().expect("vector empty!");
assert_eq!(fifth, "105");

// 方法二：与最后一个元素交换后移出
let second = v.swap_remove(1);
assert_eq!(second, "102");

// 方法三：用 std::mem::replace 替换后移出
let third = std::mem::replace(&mut v[2], "substitute".to_string());
assert_eq!(third, "103");
assert_eq!(v, vec!["101", "104", "substitute"]);
```

### 利用 for 循环消耗向量

```rust
let v = vec!["liberté".to_string(),
             "égalité".to_string(),
             "fraternité".to_string()];

for mut s in v {        // v 被整个移入循环
    s.push('!');
    println!("{}", s);
}
// v 在此处不可见，已被消耗
```

### 利用 Option 实现部分移动

将字段类型改为 `Option<T>`，借助 `take()` 安全移出：

```rust
struct Person { name: Option<String>, birth: i32 }

let mut composers = Vec::new();
composers.push(Person { name: Some("Palestrina".to_string()), birth: 1525 });

let first_name = composers[0].name.take();  // 移出值，原地留下 None
assert_eq!(first_name, Some("Palestrina".to_string()));
assert_eq!(composers[0].name, None);
```

## 八、Rc 与 Arc：共享所有权

大多数 Rust 值有唯一拥有者，但在某些情况下，单个拥有者的生命周期难以满足需求——你希望值存活到所有使用者都用完为止。Rust 通过引用计数指针类型 `Rc<T>` 和 `Arc<T>` 提供共享所有权。

### 核心思想

- `Rc<T>` 和 `Arc<T>` 是在堆上分配 `T` 并附带引用计数的指针
- 克隆 `Rc<T>` 只创建另一个指向同一数据的指针并递增引用计数，**不复制 T 本身**
- 当最后一个 `Rc` 被丢弃时，引用计数归零，Rust 自动丢弃 `T`

### Rc 与 Arc 的区别

| 类型 | 说明 |
|------|------|
| `Rc<T>` | 非线程安全，更新引用计数更快，单线程场景 |
| `Arc<T>` | 原子引用计数，线程安全，多线程共享 |

若不需要跨线程共享，优先使用 `Rc`。Rust 在编译期阻止 `Rc` 跨越线程边界。

### 基本用法

```rust
use std::rc::Rc;

let s: Rc<String> = Rc::new("shirataki".to_string());
let t: Rc<String> = s.clone();
let u: Rc<String> = s.clone();
// s、t、u 都指向同一个堆上的 String

assert!(s.contains("shira"));
assert_eq!(t.find("taki"), Some(5));
```

### 不可变性与循环引用

`Rc` 指向的值默认**不可变**——Rust 的内存安全基石是确保没有"既共享又可变的"值。若需要可变性，需结合 `RefCell` 或 `Mutex` 等内部可变性类型。

使用 `Rc` 时可能出现循环引用（A 引用 B，B 引用回 A），导致双方引用计数永不为零而内存泄漏。可通过 `Weak<T>` 弱引用打破循环。

> 详细用法（弱引用、内部可变性组合、树形结构构建等）参见：[指针类型/Rc与Arc.md](../指针类型/Rc与Arc.md)

## 九、所有权与函数

函数传参和返回值也遵循所有权规则。可以通过返回元组来交还所有权。

```rust
    fn calculate_length(s: String) -> (String, usize) {
        let len = s.len();
        (s, len)  // 将所有权返回给调用者
    }

    let s = String::from("hello");
    let (s, len) = calculate_length(s);
    // s 依然可用
```

但每次都这样返回所有权很繁琐，引用与借用提供了更好的解决方案。

## 速记要点

- 每个值有且只有一个所有者
- 离开作用域时自动释放内存
- 堆上数据赋值会**移动**所有权，原变量失效
- 栈上 `Copy` 类型自动复制，两者都可用
- 使用 `clone()` 可深拷贝堆数据
- 函数传参与返回也会发生所有权转移
- 控制流中，任一分支移动后变量即未初始化，循环中需每次重新赋值
- 不能直接从集合索引中移出元素，可用 `pop`/`swap_remove`/`replace`/`take` 等替代
- 需要共享所有权时使用 `Rc`（单线程）或 `Arc`（多线程），但数据默认不可变
