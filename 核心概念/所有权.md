# Rust 所有权

所有权是 Rust 最独特的特性，它在编译时管理内存，无需垃圾回收器或手动分配/释放。理解所有权的三个核心规则，是写出安全、高效 Rust 代码的基础。

## 一、所有权规则速览

- Rust 中每一个值都有一个被称为**所有者**的变量
- 值在任一时刻**只能有一个所有者**
- 当所有者离开作用域，值将被**自动丢弃**（调用 `drop`）

## 二、栈与堆：所有权管理的内存基础

所有权系统的设计直接针对堆内存的管理。数据存储在栈还是堆上，决定了它在所有权转移时的行为。

- **栈**：存放大小编译时已知的固定大小数据（如整数、浮点、布尔、定长元组）。栈上数据的拷贝非常廉价，直接按位复制。
- **堆**：存放大小编译时未知或可能变化的数据（如 `String`、`Vec<T>`、`Box<T>`）。堆上数据通过指针访问，拷贝代价高，所有权转移涉及指针的移动而非数据复制。

```rust
    // 栈上数据：简单拷贝
    let x = 5;
    let y = x;  // 值被完整复制到 y，x 仍然可用
    println!("x = {}, y = {}", x, y);  // 合法

    // 堆上数据：所有权移动
    let s1 = String::from("hello");
    let s2 = s1;  // s1 的堆指针被移动到 s2，s1 不再有效
    // println!("{}", s1);  // 编译错误：s1 已被移动
```

## 三、移动 (Move)：所有权的转移

当变量赋值、传递给函数、或从函数返回时，所有权可能会转移。

```rust
    // 赋值时的移动
    let s1 = String::from("hello");
    let s2 = s1;          // s1 移动到 s2
    // s1 已失效

    // 函数传参时的移动
    fn take_ownership(s: String) {
        println!("{}", s);
    } // s 离开作用域，内存释放

    let s3 = String::from("world");
    take_ownership(s3);   // s3 移动到函数参数
    // s3 已失效
```

## 四、克隆 (Clone)：深拷贝堆数据

如果需要保留原值并产生一份独立的堆数据，需要使用 `clone()` 方法进行深度复制。

```rust
    let s1 = String::from("hello");
    let s2 = s1.clone();  // 堆数据被复制，s1 和 s2 都可用
    println!("s1 = {}, s2 = {}", s1, s2);  // 合法
```

## 五、Copy trait：栈上数据的自动拷贝

拥有 `Copy` trait 的类型在赋值时自动按位复制，不会发生所有权转移。通常存储在栈上的基本类型都实现了 `Copy`，如整数、浮点、布尔、字符、元组（若其所有元素均实现 `Copy`）等。

```rust
    let x = 42;
    let y = x;  // 自动复制，非移动
    println!("{} {}", x, y);  // 两者均可用
```

## 六、所有权与函数

函数传参和返回值也遵循所有权规则。可以通过返回元组来交还所有权。

```rust
    fn calculate_length(s: String) -> (String, usize) {
        let len = s.len();
        (s, len)  // 将所有权返回给调用者
    }

    let s = String::from("hello");
    let (s, len) = calculate_length(s);
    // s 依然可用
```

但每次都这样返回所有权很繁琐，引用与借用提供了更好的解决方案。

## 速记要点

- 每个值有且只有一个所有者
- 离开作用域时自动释放内存
- 堆上数据赋值会**移动**所有权，原变量失效
- 栈上 `Copy` 类型自动复制，两者都可用
- 使用 `clone()` 可深拷贝堆数据
- 函数传参与返回也会发生所有权转移
