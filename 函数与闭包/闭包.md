# 闭包

闭包（Closure）是匿名函数表达式，能捕获其所在作用域中的变量。Rust 闭包的设计目标：比函数指针更快、零成本抽象、可在热点代码中使用。

---

## 一、闭包基础

```rust
fn sort_cities(cities: &mut Vec<City>) {
    cities.sort_by_key(|city| -city.population);  // 闭包
}

// 等效的具名函数写法：
fn city_population_descending(city: &City) -> i64 { -city.population }
cities.sort_by_key(city_population_descending);
```

Rust 从闭包的使用方式推断参数类型和返回类型。常见使用场景：
- `Iterator` 方法（`map`、`filter` 等）
- 线程 API（`thread::spawn`）
- 惰性计算默认值（`HashMap::or_insert_with`）

---

## 二、捕获变量

闭包可以"捕获"所在函数的数据。Rust 没有垃圾回收，使用**生命周期**确保安全。

### 2.1 借用值的闭包（默认）

```rust
fn sort_by_statistic(cities: &mut Vec<City>, stat: Statistic) {
    cities.sort_by_key(|city| -city.get_statistic(stat));
    // 闭包自动借入对 stat 的引用
}
```

闭包同样遵循借用和生命周期的规则——Rust 不会让闭包的生命周期超出被借用变量。

### 2.2 "窃取"值的闭包（move）

当闭包需要"逃逸"出当前函数时（如传给另一个线程），必须使用 `move` 关键字：

```rust
fn start_sorting_thread(mut cities: Vec<City>, stat: Statistic)
    -> thread::JoinHandle<Vec<City>>
{
    let key_fn = move |city: &City| -> i64 { -city.get_statistic(stat) };

    thread::spawn(move || {
        cities.sort_by_key(key_fn);
        cities
    })
}
```

`move` 闭包的规则：
- 如果捕获的是 `Copy` 类型，会**复制**该值
- 如果捕获的是非 `Copy` 类型，会**移动**该值（之后不能再使用原变量）
- 需要保留原变量时，先克隆再让闭包"窃取"副本

## 三、函数类型 vs 闭包类型

### 3.1 函数类型 `fn`

```rust
fn city_population_descending(city: &City) -> i64 { -city.population }
// 类型：fn(&City) -> i64

let my_key_fn: fn(&City) -> i64 = city_population_descending;
```

`fn` 值就是函数机器码的内存地址，类似 C++ 函数指针。

### 3.2 闭包不是 `fn`

```rust
fn count_selected_cities(cities: &Vec<City>, test_fn: fn(&City) -> bool) -> usize { ... }

let limit = 0.5;
count_selected_cities(&my_cities, |city| city.risk > limit);  // 错误！类型不匹配
```

**每个闭包都有自己的独特类型**，因为闭包需要存储捕获的数据。解决方案：使用 `Fn` 特型使函数泛型化：

```rust
fn count_selected_cities<F>(cities: &Vec<City>, test_fn: F) -> usize
    where F: Fn(&City) -> bool
{ ... }

// 现在同时接受函数和闭包
count_selected_cities(&my_cities, has_monster_attacks);        // 函数
count_selected_cities(&my_cities, |city| city.risk > limit);   // 闭包
```

## 四、三种闭包特型

### 4.1 FnOnce、FnMut、Fn 的层次关系

```
Fn : FnMut : FnOnce
```

- **`Fn`**：可不受限制地调用任意多次（通过共享引用调用）
- **`FnMut`**：可多次调用，会修改捕获的变量（通过可变引用调用）
- **`FnOnce`**：只能调用一次，会消耗捕获的值（按值调用，消耗 self）

### 4.2 伪代码定义（无参数版本）

```rust
trait Fn() -> R {
    fn call(&self) -> R;
}

trait FnMut() -> R {
    fn call_mut(&mut self) -> R;
}

trait FnOnce() -> R {
    fn call_once(self) -> R;
}
```

闭包调用 `closure()` 是这些 `call` 方法的简写。

### 4.3 闭包分类规则

| 闭包行为 | 类别 |
|----------|------|
| 只读取捕获的变量（共享引用） | `Fn` |
| 修改捕获的变量（可变引用） | `FnMut`（也是 `FnOnce`） |
| 消耗/丢弃捕获的变量（移动所有权） | `FnOnce`（仅此一种） |

### 4.4 FnOnce 示例

```rust
let my_str = "hello".to_string();
let f = || drop(my_str);  // 此闭包是 FnOnce
f();  // OK
f();  // 错误：使用了已移动的值
```

```rust
fn call_twice<F: Fn()>(closure: F) { closure(); closure(); }

let f = || drop(my_str);
call_twice(f);  // 编译错误：expected Fn, but this closure only implements FnOnce
```

迭代消耗值的常见陷阱：

```rust
let dict = produce_glossary();
let debug_dump_dict = || {
    for (key, value) in dict { ... }  // 错误：消耗了 dict！
};

// 修复：通过引用遍历
let debug_dump_dict = || {
    for (key, value) in &dict { ... }  // OK，现在是 Fn
};
```

### 4.5 FnMut 示例

```rust
let mut i = 0;
let incr = || {
    i += 1;  // 借入了对 i 的可变引用
    println!("Ding! i is now: {}", i);
};

// call_twice 需要声明为 FnMut：
fn call_twice<F: FnMut()>(mut closure: F) {
    closure();
    closure();
}
call_twice(incr);  // OK
```

> **最佳实践**：编写以闭包为参数的泛型函数时，使用能力最弱的特型（`FnMut` 优于 `Fn`，`FnOnce` 优于 `FnMut`），以接受最广泛类型的闭包。

## 五、闭包的 Copy 与 Clone

闭包的 `Copy`/`Clone` 规则与常规结构体一样：

```rust
let y = 10;
let add_y = |x| x + y;
let copy_of_add_y = add_y;                // 闭包是 Copy，因为只持有共享引用
assert_eq!(add_y(copy_of_add_y(22)), 42);
```

```rust
let mut x = 0;
let mut add_to_x = |n| { x += n; x };
let copy_of_add_to_x = add_to_x;          // 移动！可变引用不可 Copy
```

`move` 闭包：捕获的全部内容都能 `Copy`/`Clone`，闭包就能 `Copy`/`Clone`。

```rust
let mut greeting = String::from("Hello, ");
let greet = move |name| {
    greeting.push_str(name);
    println!("{}", greeting);
};
greet.clone()("Alfred");  // 克隆后调用
greet.clone()("Bruce");
```

## 六、闭包性能

Rust 闭包是**零成本抽象**：

- **没有垃圾回收**：不堆分配（除非放入 `Box`/`Vec` 等容器）
- **可以内联**：每个闭包有独特类型，编译器知道调用哪段代码，可以直接内联
- **内存紧凑**：闭包在内存中像小型结构体——`move` 闭包包含值，非 `move` 闭包包含引用
- 不捕获任何变量的闭包是空结构体，占用零内存

## 七、回调模式

### 7.1 存储不同类型的闭包：特型对象

```rust
type BoxedCallback = Box<dyn Fn(&Request) -> Response>;

struct BasicRouter {
    routes: HashMap<String, BoxedCallback>
}

impl BasicRouter {
    fn add_route<C>(&mut self, url: &str, callback: C)
        where C: Fn(&Request) -> Response + 'static
    {
        self.routes.insert(url.to_string(), Box::new(callback));
    }
}
```

注意需要 `'static` 限界——闭包不能包含对即将超出作用域的变量的借用引用。

### 7.2 使用函数指针 `fn`

如果回调不需要捕获变量，可用 `fn` 类型避免堆分配：

```rust
struct FnPointerRouter {
    routes: HashMap<String, fn(&Request) -> Response>
}
// fn 值只占一个 usize，无需 Box
```

非捕获型闭包可以退化为函数指针：`let closure_ptr: fn(u32) -> u32 = |x| x + 1;`

## 八、设计思考

Rust 没有垃圾回收，所有权必须明晰。Java/C#/JavaScript 中常见的对象相互引用的模式（如 MVC 中各组件互持引用）在 Rust 中不直接适用。替代方案：

- 让闭包接受所需引用作为参数
- 使用编号/ID 代替引用
- 采用单向数据流架构（如 Flux）替代循环引用
