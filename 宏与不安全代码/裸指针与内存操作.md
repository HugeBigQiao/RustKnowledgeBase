# 裸指针与内存操作

> 前置知识：unsafe 的核心概念（不安全块/函数、契约模型、未定义行为清单、不安全特型）请先阅读 [不安全代码基础](不安全代码基础.md)。

基于《Rust程序设计（第2版）》第22章。

---

## 1. 裸指针

裸指针在 Rust 中是一种**不受约束的指针**，只能在 `unsafe` 块中对它们解引用。裸指针本质上等效于 C/C++ 指针。

**两种裸指针类型**：

| 类型 | 含义 |
|------|------|
| `*mut T` | 指向 `T` 的允许修改其引用目标的裸指针 |
| `*const T` | 指向 `T` 的只允许读取其引用目标的裸指针 |

（没有单纯的 `*T` 类型，必须始终指定 `const` 或 `mut`。）

**创建与解引用**：

```rust
let mut x = 10;
let ptr_x = &mut x as *mut i32;

let y = Box::new(20);
let ptr_y = &*y as *const i32;

unsafe {
    *ptr_x += *ptr_y;
}
assert_eq!(x, 30);
```

**裸指针与安全指针的关键区别**：

- 裸指针**可以为空**（`std::ptr::null()` / `std::ptr::null_mut()`）
- 创建裸指针、传递裸指针和比较裸指针**都是安全的**，只有**解引用裸指针是不安全的**
- 指向无固定大小类型的裸指针是**胖指针**（`*const [u8]` 包括长度和地址，`*mut dyn Write` 携带虚表）
- **`.` 运算符不会隐式解引用裸指针**，必须写成 `(*raw).field` 或 `(*raw).method(...)`
- 裸指针没有实现 `Deref`，因此隐式解引用不适合它们
- `==`、`<` 等运算符将裸指针作为**地址**进行比较
- `Debug` 和 `Pointer` 格式化特型会将裸指针展示为**十六进制地址**

**引用 → 裸指针的隐式转换**：Rust 会将引用隐式转换成裸指针（反过来**不成立**）。

---

## 2. 安全地解引用裸指针

以下是安全使用裸指针的**常识性指南**：

1. **解引用空指针或悬空指针是未定义行为**，引用未初始化的内存或超出作用域的值也一样。
2. **解引用未正确对齐的指针是未定义行为。**
3. **只有在遵守引用安全规则的前提下**，才能从解引用的裸指针中借用值（任何引用的生命周期都不能超出其引用目标，共享访问是只读访问，可变访问是独占访问）。
4. **仅当引用目标是所属类型的格式良好的值时**，才能使用裸指针的引用目标（例如，`*const char` 必须是正确的 Unicode 码点）。
5. 使用 `offset` 和 `wrapping_offset` 时，裸指针**只能指向原初指针所引用的变量内部的字节或分配在堆上的内存块内部的字节**，或者指向上述两个区域之外的**第一字节**。
6. 如果要给裸指针的引用目标赋值，**不得违反引用目标所属的任何类型的不变条件**（例如，指向 `String` 中一字节的 `*mut u8` 不能存储非 UTF-8 字节）。

## 3. 示例：`RefWithFlag`（位级 hack 的安全包装）

利用对齐特性，将 `&T` 和 `bool` 存储在**单个机器字**中：

```rust
mod ref_with_flag {
    use std::marker::PhantomData;
    use std::mem::align_of;

    pub struct RefWithFlag<'a, T> {
        ptr_and_bit: usize,
        behaves_like: PhantomData<&'a T> // 不占空间
    }

    impl<'a, T: 'a> RefWithFlag<'a, T> {
        pub fn new(ptr: &'a T, flag: bool) -> RefWithFlag<T> {
            assert!(align_of::<T>() % 2 == 0);
            RefWithFlag {
                ptr_and_bit: ptr as *const T as usize | flag as usize,
                behaves_like: PhantomData
            }
        }

        pub fn get_ref(&self) -> &'a T {
            unsafe {
                let ptr = (self.ptr_and_bit & !1) as *const T;
                &*ptr
            }
        }

        pub fn get_flag(&self) -> bool {
            self.ptr_and_bit & 1 != 0
        }
    }
}
```

**核心技巧**：许多类型必须放置在**偶数地址**（最低有效位始终为 0），因此可以在最低位存储 `bool`，通过屏蔽最低位来重建原始地址。

**`PhantomData` 的作用**：包含 `PhantomData<&'a T>` 字段告诉 Rust 应该将 `RefWithFlag<'a, T>` **视为**包含一个 `&'a T`，从而正确约束生命周期，却不会实际影响结构体的表示方式。

## 4. 可空指针

Rust 中的空裸指针是一个零地址：

```rust
// 创建空指针
let p: *const T = std::ptr::null::<T>();
let p: *mut T = std::ptr::null_mut::<T>();

// 检查是否为空
if ptr.is_null() { ... }

// 转换为 Option<&T> / Option<&mut T>
let opt_ref: Option<&T> = ptr.as_ref();       // 空指针 → None
let opt_mut: Option<&mut T> = ptr.as_mut();   // 空指针 → None
```

## 5. 类型大小与对齐方式

```rust
// 固定大小类型的大小和对齐方式
assert_eq!(std::mem::size_of::<i64>(), 8);
assert_eq!(std::mem::align_of::<(i32, i32)>(), 4);

// 任何类型总是对齐到 2 的 n 次幂
// 类型的大小总是会四舍五入为其对齐方式的倍数
// 例如 (f32, u8) 只需要 5 字节，但 size_of 返回 8

// 无固定大小类型的值大小和对齐方式
let slice: &[i32] = &[1, 3, 9, 27, 81];
assert_eq!(std::mem::size_of_val(slice), 20);

let text: &str = "alligator";
assert_eq!(std::mem::size_of_val(text), 9);
```

## 6. 指针运算

Rust 将数组、切片或向量的元素排布为**单个连续的内存块**，元素间隔均匀。

```rust
// offset：以元素为单位的指针偏移
let trucks = vec!["garbage truck", "dump truck", "moonstruck"];
let first: *const &str = &trucks[0];
let last: *const &str = &trucks[2];
assert_eq!(unsafe { last.offset_from(first) }, 2);
assert_eq!(unsafe { first.offset_from(last) }, -2);
```

**重要规则**：

- 可以生成指向数组末尾之后第一字节的指针（用于边界检查），但不能对其解引用
- 使用 `offset` 生成超出该点或指向数组开头之前的指针是**未定义行为**，即使从未解引用
- 如果确实需要偏移到关联数组界限之外，使用 **`wrapping_offset`**——Rust 不会假设相对顺序，但仍然不能解引用

## 7. 移动入和移动出内存（`ptr::read` / `ptr::write`）

Rust 为实现管理自己内存的类型提供了两个基本操作：

| 函数 | 说明 |
|------|------|
| `std::ptr::read(src)` | 将值**移动出** `src` 指向的位置，将所有权转移给调用者。调用后 `*src` 应视为未初始化内存（除非 `T: Copy`）。这是 `Vec::pop` 背后的操作。 |
| `std::ptr::write(dest, value)` | 将 `value` **移动入** `dest` 指向的位置，该位置在调用前必须是未初始化内存。这是 `Vec::push` 背后的操作。 |

**批量移动**：

| 函数 | 说明 |
|------|------|
| `std::ptr::copy(src, dst, count)` | 将 `count` 个值的数组从 `src` 移动到 `dst`（允许重叠） |
| `std::ptr::copy_nonoverlapping(src, dst, count)` | 非重叠版本的批量移动（可能更快） |

> 不能使用任何 Rust 的安全指针类型来执行这些操作。安全指针要求其引用目标始终是初始化的，而裸指针没有这种限制。

**初始化值的真正定义**：应视为**有效**的值。写入值的字节通常是初始化的必要部分，但这只是为了将其视为有效而做的准备工作。

## 8. 示例：`GapBuffer`（间隙缓冲区）

间隙缓冲区是一种在**恒定时间内**进行插入和删除的数据结构，常用于文本编辑器（如 Emacs）。

**核心思想**：将空闲容量（间隙）保留在文本**中间**（正在编辑的位置），而不是末尾。

```rust
pub struct GapBuffer<T> {
    // 元素的存储区（长度始终保持为 0，通过裸指针直接使用其容量）
    storage: Vec<T>,
    // storage 中间未初始化元素的范围
    gap: Range<usize>
}
```

**关键方法**：

- `set_position`：使用 `std::ptr::copy` 将间隙移动到新位置
- `insert`：使用 `std::ptr::write` 在间隙处写入元素
- `remove`：使用 `std::ptr::read` 从间隙后读取元素
- `enlarge_gap`：使用 `std::ptr::copy_nonoverlapping` 扩容
- `Drop`：使用 `std::ptr::drop_in_place` 丢弃间隙前后的所有元素

**设计原则**：`GapBuffer` 会确保自己的不变条件足以遵守所使用的每个不安全特性的契约，因此它的所有公共方法都不需要标记为 `unsafe`。

## 9. 不安全代码中的 Panic 安全性

在 `unsafe` 代码中，panic 可能导致**未定义行为**。

```rust
pub fn remove(&mut self) -> Option<T> {
    // ...
    let element = unsafe {
        std::ptr::read(self.space(self.gap.end))  // ← 此时 GapBuffer 处于不一致状态
    };
    // 如果这里 panic，间隙外会留下未初始化的元素 → 未定义行为！
    self.gap.end += 1;
    Some(element)
}
```

**关键原则**：
- 类型的方法在执行工作时几乎不可避免地会**暂时放松类型的不变条件**，然后在返回之前让其回到正轨
- 方法中间出现的 **panic 可能会中断清理过程**，使类型处于不一致状态
- 如果类型使用了不安全特性，破坏不变条件会导致**契约破损 → 未定义行为**
- 必须**特别注意识别那些暂时放松了不变条件的敏感代码区域**，并确保它们不会执行任何可能引起 panic 的事情

## 10. 联合体（Union）

联合体是 Rust 用于**重新解释内存**的强大特性：

```rust
union FloatOrInt {
    f: f32,
    i: i32,
}

let mut one = FloatOrInt { i: 1 };
assert_eq!(unsafe { one.i }, 0x00_00_00_01);
one.f = 1.0;
assert_eq!(unsafe { one.i }, 0x3F_80_00_00);  // IEEE 754 浮点数编码
```

**联合体的关键特性**：

- 联合体的字段会引用**相同位序列的不同解释**
- 联合体的**大小由其最大字段决定**
- **构造联合体或对其字段赋值是完全安全的**
- **读取联合体的任何字段都是不安全的**（因为没有标签来区分变体）
- 所有字段都必须是 `Copy` 的。如需存储非 `Copy` 类型，使用 `std::mem::ManuallyDrop`

**`#[repr(C)]` 与联合体**：添加 `#[repr(C)]` 可以保证所有字段都从偏移量 `0` 开始，适用于跨 FFI 使用和精确的位级操作：

```rust
#[repr(C)]
union SignExtractor {
    value: i64,
    bytes: [u8; 8]
}

fn sign(int: i64) -> bool {
    let se = SignExtractor { value: int };
    unsafe { se.bytes[7] >= 0b10000000 }  // 最高有效字节的最高有效位
}
```

## 11. 匹配联合体

在 Rust 联合体上进行匹配和在结构体上匹配类似，但每个模式**必须指定一个字段**：

```rust
unsafe {
    match u {
        SmallOrLarge { s: true } => { println!("boolean true"); }
        SmallOrLarge { l: 2 } => { println!("integer 2"); }
        _ => { println!("something else"); }
    }
}
```

> 与联合体变体匹配但不指定值的 `match` 分支**永远都会成功**。

> 如果最后写入的字段与匹配的字段不同，会导致**未定义行为**。

---

## 12. 借用联合体

借用联合体的一个字段就是**借用整个联合体**：

- 将一个字段作为**可变借用**会排斥对该字段或其他字段的任何其他借用
- 将一个字段作为**不可变借用**意味着对任何字段都不能再进行可变借用

## 13. FFI 安全包装（概述）

通过 Rust 的外部函数接口（FFI），可以在 Rust 中调用 C/C++ 函数。`extern` 块声明要链接的外部函数：

```rust
use std::os::raw::c_char;

#[link(name = "git2")]
extern {
    pub fn git_libgit2_init() -> c_int;
    pub fn git_libgit2_shutdown() -> c_int;
}
```

**FFI 核心要素**：

- **`#[repr(C)]`**：保证结构体/枚举的内存布局与 C 兼容
- **`std::os::raw` 类型**：`c_int`、`c_char`、`c_void` 等，保证与 C 类型具有相同表示
- **不完整结构体**：`#[repr(C)] pub struct git_repository { _private: [u8; 0] }` 映射 C 的不透明类型
- **C 字符串**：`std::ffi::CString`（拥有型）和 `CStr`（借用型），以空字符结尾
- **`extern` 块中的所有函数都是 `unsafe` 的**
- **构建脚本（build.rs）**：通过 `cargo:rustc-link-search` 等指令告诉 Cargo 在哪里查找库

> **`-sys` crate 约定**：提供 C 库访问能力的 crate 应命名为 `LIB-sys`，只包含静态链接库、`extern` 块和类型定义。更高级别的安全接口属于依赖 `-sys` crate 的独立 crate。
