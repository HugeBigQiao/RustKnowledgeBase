# Rust 宏

## 一、概述

**宏**是 Rust 中扩展语言的方式，能完成函数无法做到的事。宏在编译早期被**展开**（替换为 Rust 代码），类型检查之前就已完成。

与 C++ 宏的区别：
- 宏调用带有 `!`，易于识别
- 不会插入不匹配的括号
- 天生支持模式匹配
- 支持**卫生宏**（不会意外干扰局部变量）

---

## 二、`macro_rules!` 基础

### 2.1 基本语法

```rust
macro_rules! assert_eq {
    ($left:expr, $right:expr) => {
        match (&$left, &$right) {
            (left_val, right_val) => {
                if !(*left_val == *right_val) {
                    panic!("assertion failed: `(left == right)`, \
                            (left: `{:?}`, right: `{:?}`)", left_val, right_val);
                }
            }
        }
    };
}
```

宏主体是一系列规则：`( pattern ) => ( template );`

### 2.2 模式匹配与展开

- **模式**用于匹配宏参数（操作的是语法标记 token，而非字符）
- **模板**中 `$name` 会被替换为匹配到的代码片段
- 注释和空白在模式中被忽略（它们不是语法标记）

### 2.3 宏展开的顺序

Rust 从头到尾阅读源码，定义并展开宏。**不能在定义宏之前调用它**（与函数不同）。

---

## 三、宏的注意事项

### 3.1 避免副作用

宏只计算一次表达式并存储值：

```rust
// 正确：借用引用，防止移动
match (&$left, &$right) { ... }

// 错误：会移动 String
match ($left, $right) { ... }
```

### 3.2 模板中常见错误

在模板中写成 `$left:expr` 而非 `$left` 会导致编译错误（`:expr` 会被当成普通语法标记输出）。

## 四、重复模式

### 4.1 `vec!` 宏示例

```rust
macro_rules! vec {
    ($elem:expr ; $n:expr) => {
        ::std::vec::from_elem($elem, $n)
    };
    ( $( $x:expr ),* ) => {
        <[_]>::into_vec(Box::new([ $( $x ),* ]))
    };
    ( $( $x:expr ),+ ,) => {
        vec![ $( $x ),* ]  // 支持尾随逗号
    };
}
```

### 4.2 重复模式表

| 模式 | 含义 |
|------|------|
| `$( ... )*` | 匹配 0 次或多次，无分隔符 |
| `$( ... ),*` | 匹配 0 次或多次，逗号分隔 |
| `$( ... );*` | 匹配 0 次或多次，分号分隔 |
| `$( ... )+` | 匹配 1 次或多次，无分隔符 |
| `$( ... ),+` | 匹配 1 次或多次，逗号分隔 |
| `$( ... );+` | 匹配 1 次或多次，分号分隔 |
| `$( ... )?` | 匹配 0 次或 1 次，无分隔符 |

### 4.3 模板中使用重复

```rust
// 模式中捕获的 $x 是表达式列表
// 模板中同样使用 $( ... ),* 展开
<[_]>::into_vec(Box::new([ $( $x ),* ]))
```

## 五、片段类型

`macro_rules!` 支持的片段类型：

| 类型 | 匹配内容 | 后可跟的标记 |
|------|----------|-------------|
| `expr` | 表达式：`2+2`、`x.len()` | `=>`, `;` |
| `stmt` | 语句 | `=>`, `;` |
| `ty` | 类型：`String`、`Vec<u8>` | `=>;=|\{[`:>as where` |
| `path` | 路径：`ferns`、`::std::sync` | `=>;=|\{[`:>as where` |
| `pat` | 模式：`_`、`Some(ref x)` | `=>`, `=\|if in` |
| `item` | 语法项：`struct`、`mod` | 任意 |
| `block` | 块：`{ ... }` | 任意 |
| `meta` | 属性主体：`inline`、`derive(...)` | 任意 |
| `literal` | 字面量：`1024`、`"hello"` | 任意 |
| `lifetime` | 生命周期：`'a`、`'static` | 任意 |
| `vis` | 可见性：`pub`、`pub(crate)` | 任意 |
| `ident` | 标识符：`std`、`Json` | 任意 |
| `tt` | 语法标记树：`{}`、`>=`、`[0 1 (+ 0 1)]` | 任意 |

`tt`（token tree）是关键：它能匹配成对的括号和嵌套结构，适合解析非标准 Rust 语法。

## 六、宏中的递归

### 6.1 `json!` 宏示例

```rust
macro_rules! json {
    (null) => { Json::Null };
    ([ $( $element:tt ),* ]) => {
        Json::Array(vec![ $( json!($element) ),* ])
    };
    ({ $( $key:tt : $value:tt ),* }) => {
        Json::Object(Box::new(vec![
            $( ($key.to_string(), json!($value)) ),*
        ].into_iter().collect()))
    };
    ($other:tt) => {
        Json::from($other)  // 使用 From 特型
    };
}
```

### 6.2 递归限制

默认最多递归 64 层，可通过 crate 属性调整：

```rust
#![recursion_limit = "256"]
```

## 七、卫生宏（Hygiene）

### 7.1 宏中的变量不会与调用者冲突

```rust
macro_rules! some_macro {
    () => {
        let fields = Box::new(HashMap::new());
        // ...
    }
}

let fields = "Fields, W.C.";
some_macro!();
// fields 仍然是 "Fields, W.C."——宏内部的 fields 是卫生的
```

Rust 会重命名宏引入的局部变量，使其不会与调用作用域的变量冲突。

### 7.2 卫生机制的范围

- **局部变量和参数**：有卫生保护
- **类型、方法、模块、静态值、宏名**：没有卫生保护（直接引用作用域中的名称）

### 7.3 需要在调用者和宏间共享标识符时

将标识符作为宏参数传递：

```rust
macro_rules! setup_req {
    ($req:ident, $server_socket:ident) => {
        let $req = ServerRequest::new($server_socket.session());
    }
}

setup_req!(req, server_socket);
```

## 八、导入和导出宏

### 8.1 `#[macro_use]` 导出

```rust
#[macro_use] mod macros;  // macros 模块的宏在父模块中可见
```

### 8.2 `#[macro_export]` 公开导出

```rust
#[macro_export]
macro_rules! lazy_static { ... }

// 使用方
use lazy_static::lazy_static;
```

导出的宏应使用绝对路径，使用 `$crate` 指代定义此宏的 crate 根模块：

```rust
$crate::Json::Null                    // 类型路径
$crate::macros::HashMap::new()         // 模块内重新导出的类型
$crate::macros::ToString::to_string($key)  // 完全限定方法调用
```

## 九、内置宏

| 宏 | 描述 |
|------|-------------|
| `file!()` | 展开为当前文件名（字符串字面量） |
| `line!()` | 展开为当前行号（`u32`） |
| `column!()` | 展开为当前列号（`u32`） |
| `stringify!(...tokens...)` | 将语法标记转为字符串，**不展开**内部宏调用 |
| `concat!(s0, s1, ...)` | 串联字符串字面量 |
| `cfg!(...)` | 编译期条件检查，返回 `bool` |
| `env!("VAR_NAME")` | 获取编译期环境变量的值 |
| `option_env!("VAR_NAME")` | 可选环境变量，返回 `Option<&str>` |
| `include!("file.rs")` | 包含文件内容为 Rust 代码 |
| `include_str!("file.txt")` | 包含文件内容为 `&'static str` |
| `include_bytes!("file.dat")` | 包含文件为 `&'static [u8]` |
| `todo!()` | 标记未完成代码（panic） |
| `unimplemented!()` | 标记未实现分支（panic） |
| `matches!(value, pattern)` | 模式匹配测试，返回 `bool` |

## 十、避免语法错误

当多条规则可能混淆时：
1. **每条规则以不同的标识符开头**：
```rust
(msg : $msg:expr) => { ... }      // 以 msg 开头
(user : $userid:tt , msg : ...) => { ... }  // 以 user 开头
```

2. **将更具体的规则放在前面**

## 十一、过程宏（Procedural Macros）

Rust 1.15 引入了**过程宏**，以 Rust 函数形式编写，可以与编译器交互实现任意复杂功能：

- **自定义派生宏**（`#[derive(MyTrait)]`）
- **属性宏**（`#[my_attribute]`）
- **函数式宏**（类似 `macro_rules!`）

常见应用：`serde` 的 `Serialize/Deserialize`、`diesel` 的数据库模式代码生成。

## 十二、调试宏

### 12.1 `trace_macros!`

```rust
#![feature(trace_macros)]

trace_macros!(true);
let numbers = vec![1, 2, 3];
trace_macros!(false);
```

编译器会打印每个宏调用的展开前后代码。

### 12.2 `log_syntax!()`

将参数打印到编译终端（需要 `#![feature(log_syntax)]`）。

### 12.3 `cargo rustc -- -Z unstable-options --pretty expanded`

展开所有宏并输出完整代码。
